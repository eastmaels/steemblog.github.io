在X61上安装Ubuntu 18.04 Server遇到的坑

虽然我有好多台Ubuntu 18.04 Server版本的VPS以及一台安装Ubuntu 18.04 Server本地的Intel NUC7I5BNHL小机器，但是我还是需要一台Ubuntu 18.04 Server的测试机。

虽然可以使用虚拟机等方式在本地机器上搞定，但是终归感觉不爽，恰巧手头有个X61，拿来装个Ubuntu 18.04 Server试试喽。

以前在Intel NUC7I5BNHL上装Ubuntu那是相当顺畅了，没想到在X61上安装，竟然遇到了好几个坑。

坑一：无法U盘启动

Ubuntu 18.04 Server安装U盘我直接用的之前的，为了正常启动，我还进X61 BIOS中设置了启动顺序。

可是还是直接进入机器自带的XP系统。使用F12呼出启动菜单，发现竟然没有USB的项目，会是什么问题呢？再次进BIOS中查看，发现USB项目下有如下内容：

原来需要启用BIOS对USB的支持，我以前可能手欠禁用了😓

坑二：无法进入安装

原本以为会是一路顺畅，结果安装程序启动起来一直给我提示如下内容：

gfxboot.c32: not a COM32R image

这又是什么鬼？我装Intel NUC7I5BNHL的时候没遇到过呀。查了一下说是BUG

https://bugs.launchpad.net/ubuntu/+source/usb-creator/+bug/1325801

解决方式是敲live或者install进行安装，我敲的是什么？可能是install吧。两者有啥区别，我也不知道。

坑三：无法跳过网卡设置

进入安装了，总该没问题了吧。结果半路却卡在这步，没法前进了😂

我试了N次，发现实在没有办法，只好把本本抱到路由器旁边，然后找到一根网线插上，然后才顺利继续。

尽管有很多坑，最后总算安装成功啦，不过现在装系统觉得太累了，还有我再考虑要不要给它升级个SSD啥的，然后再重新装一遍呢？愁人！
