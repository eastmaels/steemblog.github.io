不聊保险，我们还能好好做朋友

这两天李咏去世，让做体检的和做保险业务的人集体狂欢了一把，朋友圈大凡和这个相关都集体高潮了。


(图源 ：pexels.com)

更有在保险业务员煞有介事的发帖称李咏去世，保险公司赔了3亿元。然后估计发完后自己都不相信，又加了一些解释，说什么保险公司有赔付限额，李咏一定是投了好多家保险公司才拿到这么多赔付的。

好吧，就算是在多家保险公司投保，可是要靠寿险赔付这么多，估计签单就累死了吧。可惜人死不能复生，如果李咏泉下有知，估计会气得活过来，一一找他们算账吧？

其实还有另外一点，即便真的能赔付这么多，但是去除公司运营等成本，保险本质上是一个互助救济基金，你一下子从基金池中掏走这么多，那么将来别人出意外的时候，得到的自然就少了。

我有个朋友，人挺好的，平时和我关系还不错。可是这两年开始做保险，一直向我和圈子里的其它朋友推销，有的朋友被推销的烦了，就严词拒绝，然后自然朋友就没法做了。

碍于情面，我一直拒绝的比较委婉，可是这给她造成了我或许是她潜在客户的误解。这不，李咏这事刚一发生，她又不遗余力地向我推销保险了。

先是说我那么有钱一定要支持她一下，然后我表示我都穷的揭不开锅了，她又说没钱更得买了，否则要是生病了怎么办？要是去世了怎么办？😱

后来我实在是不知道咋回复好了，于是回复她：“咱以后不聊保险的事，好不好？不聊保险，我们还能好好的做朋友。”

可是显然我的话并没有起到任何作用，她继续她的一大堆为了我好的理由。想了想，我还是没忍心拉黑，只好装作视而不见了。


(图源 ：pexels.com)

话说，你有没有亲戚朋友做保险的呢？他们有没有向你推销保险？你又是如何处理的呢？

自己动手，清洗地热

据说暖气主管道漏水已经解决，现在已经开始带热水运行了，既然如此我也应该把我的地热供暖都打开喽，但是经过快一年的闲置，地热管道里肯定一堆脏水和铁锈之类的，为了让它运行良好，应该清洗一下。


(图源 ：pexels.com)

一般洗地热都是找专门的人员用专业的设备处理，当然了价格也是不菲，为了省钱，我决定自己简单操作一下。

详细步骤如下 ，如果你家也地热供暖的话，不妨一试哦。


先打开供暖供水阀。


关闭供暖回水阀。(顺便说一下，我的泵已经坏掉不转了）


管道阀门，每个阀门对应一个房间或者一块区域。先关闭所有的阀门，只开一组。


通过放水阀门排水。


管道里的脏水，看看脏成什么样子了！


大概放出4-5盆水以后，水变得干净一些了。

继续上述步骤（单独开一个区域阀门，其它关闭），直到所有区域都清洗干净。然后打开所有的区域阀门，打开供暖回水阀，搞定。

是不是很简单啊？当然了，就是太累了，我至少放出去几十盆水，累死我啦。所有有条件的还是找专业人士花钱弄吧。

每天进步一点点：PHP输出整段文本

话说，尽管坚信PHP是世界上最好的语言，但是真的好多年没怎么用PHP了。好多年前，一个上市公司出品的几款安全软件的后台、用户管理以及在线处理付款生成注册码之类的脚本全是我用PHP做的，现在PHP如果不查手册，我只会写Hello World!了。


(图源 ：pixabay)

当年我还熟练地使用Smarty以及PHPAJAX之类的东西，现在查了一下，我擦，这都是啥玩意啊，还有新出的流行模板库，看了几个感觉一头雾水。哎，除了拿一些往事吹吹牛，PHP我真的已经变成菜鸟了。

话说不想用WordPress等CMS，也不想用各种模板库(我不会用)，那么PHP和HTML混排，一大堆文本该咋输出啊？经过我努力学习，大概有以下几种方式。

HTML和PHP混排一

<?
// PHP代码
echo "Hello World!";
?>
在此处写HTML等文本代码
随便写啊，随便写
<?
// 继续PHP代码
echo "Hello World!";
?>

HTML和PHP混排二

<?
// PHP代码
echo "Hello World!";
$name="oflyhigh";
?>
在此处写HTML等文本代码
<p>你好<? echo $name ?>，吃饭了吗？</p>
随便写啊，随便写
<?
// 继续PHP代码
echo "Hello World!";
?>

使用双引号、单引号

<?
// PHP代码
echo "Hello World!";
$name="oflyhigh";
echo("在此处写HTML等文本代码
<p>你好$name，吃饭了吗？</p>
随便写啊，随便写");
// 继续PHP代码
echo "Hello World!";
?>

echo函数中字符串用双引号包围，函数中的变量会被解析，比如上述代码中$name会替换成oflyhigh；如果是单引号，变量被当成正常的字符串处理。

如果使用双引号或者单引号包围字符串而字符串中含有和包围字符相同的字符，则需要使用\转义。

Heredoc 以及 Nowdoc

Heredoc 以及 Nowdoc 和双引号以及单引号包围的字符串类似。但是因为它本身使用的是另外一种写法，所以不会和内容部分中的引号或者单引号冲突，所以可以让内容保持原样，无需转义。

<?
// PHP代码
echo "Hello World!";
$name="oflyhigh";
echo <<<"EOD"
在此处写HTML等文本代码
<p>你好$name，吃饭了吗？</p>
随便写啊，随便写，什么"以及'都可以
EOD;
// 继续PHP代码
echo "Hello World!";
?>

其中EOD啥的叫标识符，命名规则啥的和变量名一样，需要注意的是开始的标识符后边以及结束的标识符前边同一行内不能有任何内容。据说，PHP7.3 以后会更灵活，可惜我服务器PHP没升级。

不多举例啦，举多了怕出错，读者可以自己尝试😀，更多详情，看参考链接吧。
