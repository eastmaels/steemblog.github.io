指导DIY装机第一单

之前帖子中说经过这次DIY装机后，我已经变成了装机小能手，完全可以靠给别人出装机方案&装机赚钱啦，毕竟京东上装机服务要150元左右呢。

不过没想到第一单到的这么快，朋友看到我装机后，说自己也想换一套电脑，让我帮忙出个主意。

考虑他不打游戏、不做绘图渲染之类工作，也不跑什么大的计算程序，我建议他选购Intel的NUC，并且选择八代I5的版本就好了。

朋友觉得在网上看图片没有太直观的感受，特意来我这里参观了一下我的NUC，一下子就喜欢上了，断然决定就买这个啦。

NUC是准系统，只需要额外购买并加装内存和硬盘就可以了，内存我推荐他用32G的金士顿内存，硬盘我推荐他用三星970 PRO 1T的。

然而朋友鸡贼的狠，说什么也不肯上32G内存和970 RPO，并且用TWB给我算了一下SSD的使用寿命，按他的使用频度，估计可以用几百年。

好吧，我败下来了，最终朋友选择了16G的金士顿骇客神条，以及500G的三星970 EVO PlusM.2接口的SSD。

装好系统后，实测7秒开机，朋友满意得不得了，一直说比他之前的电脑强多啦。

还有就是原本他计划组装新电脑后，将老电脑淘汰给他的岳父，可是接触这个NUC后，发现这个NUC简直太完美并且不占空间还省电，于是决定把老电脑卖废品，给岳父也配一个NUC。

虽然这是我接到的装机指导第一单，不过和朋友这么熟，也不好意思提钱啊，哈哈哈，不过他和我说改天请我吃饭表示感谢，反正赚钱的目的也是为了吃饭，那么也算是成功啦。

以讹传讹？ fdisk到底支不支持2T以上容量的硬盘分区？

买了个大硬盘，分区时才想起来这是我第一次使用容量超过2TB的硬盘呢，而据说MBR方式不支持超过2T的分区，那么又该分区呢？


(图源 ：pixabay)

我百度查了一下，一般都是磁盘类型用gpt，分区工具用parted，不过fdisk真的不能胜任吗？可是毕竟fdisk我熟啊，撸起袖子直接试吧。

fdisk分区大硬盘

首先查看一下分区前的磁盘。

sudo fdisk -l

返回信息如下：

使用fdisk直接开干：

sudo fdisk /dev/sda

返回信息如下：

其中红色高亮部分：

The size of this disk is 3.7 TiB (4000787030016 bytes). DOS partition table format cannot be used on drives for volumes larger than 2199023255040 bytes for 512-byte sectors. Use GUID partition table format (GPT).

翻译过来就是：

这个磁盘的大小是3.7 TiB(4000787030016字节)。对于512字节扇区，DOS分区表格式不能用于大于2199023255040字节的驱动器。使用GUID分区表格式(GPT)。

直接输入p开始分区试试

可是提示中显示的还是dos类型

Disklabel type: dos
Disk identifier: 0x6b16bef3

输入m看看帮助，注意其中我圈上的部分：

输入g创建新的空白GPT分区表

返回如下：

注意这部分：

Disklabel type: gpt
Disk identifier: 5FC9A2DF-669E-7A4B-8302-157081750205

可见分区表已经修改为GPT方式啦，剩下的事情就简单啦，一路默认就可以了。之后再用sudo mkfs.ext4 /dev/sda1格式化，再编辑/etc/fstab来自动挂载就行啦，此处就不再赘述啦。

挂载后看看空间，一切正常：

以讹传讹

在网上查找了一下，无论是中文还是英文站点，很多页面中的内容还是说fdisk不支持2T以上分区，必须用parted这个工具。

造成这个问题我猜有几个因素

  • 旧版本fdisk确实不支持2T以上分区
  • 可能有些朋友用fdisk还是旧的版本
  • 直接拿前人的经验，而并没有去尝试

如果不是去尝试一下，我也和其它人一样坚信fdisk不支持2T以上分区呢。

请叫我DIY小能手

在组装这个大电脑之前，我甚至从未想到自己真的可以独立完成组装电脑的全部过程，毕竟除了自己的第一台PC是在装机店购买的，之后用的全是品牌机、笔记本和准系统。

今天硬盘到货了，因为大电脑有两个硬盘位，所以我又找朋友要来一块500G的硬盘，用来占位，这样就可以避免灰尘从底部进入机箱啦。

把两块硬盘装上之后，装机的过程应该算是彻底完成了，这期间最折磨人的莫过于主板瑕疵换掉一次，和Linux系统不兼容，折腾了几回，其它的磕磕绊绊总算都过来了。

想象中的最坏的情况，主板冒烟、CPU被烧、SSD无法读写数据之类的，都没有发生，所以总体算是顺利的啦。唯一郁闷的是京东上好多硬件都降价了，然而我这个已经过了保价期。😭

顺道秀一下我的银河硬盘

海盗船这个机箱的两个SATA硬盘位设计成插拔式，还是非常方便的，把硬盘装在托架上，往里一堆，咔嗒一声响，那声音简直不要太美妙。

装机之后fdisk看一下：

哈哈，我朋友的硬盘里还有好几个分区呢，会不会有比特币钱包呢？想想连比特币这个词汇他都是从我这里听说，我就放弃不切实际的幻想了。

反正去他那拆机的时候已经和他确认过了硬盘里没有重要数据啦，果断重新分区啦。

详细操作见文末相关链接啦，都是大同小异。

话说我觉得我又掌握了一项全新的技能，装机啥的完全没问题了，以后STEEM混不下去，我就出去给别人装机好喽。

买了一块企业级硬盘

我给我的大电脑配了两块硬盘，分别是512GB的Intel 760P NVMe M.2 SSD以及2TB的Intel 760P NVMe M.2 SSD，一块用作系统一块用作工作区。


(图源：京东商品页面截图)

可是系统都弄好之后，我的强迫症又范了，万一这两块硬盘坏掉可咋办？好吧，系统盘坏掉其实也无所谓买个新盘替换一下重装系统就好了，可以万一工作盘坏掉了，可咋办呢？

其实我一直长草的都是群晖的NAS，然而算了一下，便宜点的DS718+ 2盘位 NAS也要3600元，成本还是有些高，再加上硬盘，这价格让人难以接受。

所以最经济的方案还是加一块硬盘啦，但是硬盘加哪个品牌的、加哪个型号的、加多大容量的，又开始纠结起来。

首先桌面级硬盘肯定要排除掉，尽管我觉得桌面级硬盘7x24小时工作也完全没有问题，比如我现在一台笔记本就是常年不关机，也没见硬盘如何？但是觉得贵一点的总会靠谱一些吧？

选来选去，就剩下西数红盘和希捷的企业级硬盘看起来还可以，红盘用的人比较多，但是我承认我是个名字控，希捷的企业硬盘名字叫“希捷银河Exos”，看起来真的很酷。

至于容量嘛，我觉得2T有些小，8T刚刚好，但是8T太贵了啊，所以折中选择个4T的好啦，钱包不鼓，作啥都底气不足。


(图源：京东商品页面截图)

这个硬盘我下单时价格是988元，现在涨到了1029元，不过多半天的时间，京东这价格改来改去，也是醉了。

为了避免被认为是京东的托，链接我就不贴啦，哈哈。

发大水了：暖气主杠爆裂

这几天热电厂已经开始供暖，家里室温高达二十六七度，心里还挺美的，今年热电厂烧得不错。


(图源 ：pixabay)

上午在院子里忙活的时候，看到两个热电厂的员工，还和他们打个招呼，告诉我去楼上看看管道渗漏的问题，我心说还好只是渗漏，否则我又要遭殃了。

中午到了吃饭时间，听到楼上砸墙的声音，我也没注意，他们检查管道，肯定要开个口子看看。我们正安心地坐在餐桌前吃饭，突然发现一股股清流顺着厨房北侧的墙砖流了下来。

擦，发大水了，我们赶紧找出拖把以及水盆开始清理积水，然而眼看着水流越来越大啊，我决定上楼看看发生了什么事情。

到了楼上楼梯过道一看，我擦，过道里积满了水，足足有三五厘米那么深，而滚滚洪流还顺着楼上的楼梯哗哗淌下来。

发生了什么事？我又遇到那两个热电厂员工，结果这俩人言辞闪烁地和我说暖气主杠爆管了。我擦，你们不是去检查渗漏去了嘛？怎么还把主杠干爆了？

这俩员工死活不承认是他们砸爆的，说本来就要爆了，只是他们恰好砸开。我擦，我信你的鬼。

不过主杠已经爆管了，你们不会赶紧把阀门关上吗？可是丫和说，阀门井那有一个车辆压在井盖上，没法关阀门，不过正在联系。晕，等你们联系完，我家岂不是彻底变水帘洞了吗？

没办法我只好先联系物业和管家让他们赶紧派一些保洁员来清理积水，否则一会这些水全泡进我的房间里啦。

过了一会物业派过来五六个保洁员，带着拖布、大水桶、扫帚等工具，开始了抢险，把积在楼道里的水统统排出去。

等弄得差不多，再回家看，发现墙上水流变小多了，变成慢慢的渗漏了，不过不光厨房的墙壁，餐厅侧面的墙壁也开始渗水了。


(图源 ：pixabay)

又是擦地，又是擦墙，忙了大半天，总算好一些了，哎，真是郁闷啊。

升级本地EOS节点至v1.8.5 & CPU挖矿？

Blockone在15天以前发布了EOSIO v1.8.5，据说解决了Net plugin的一些问题，并且加入Subjective CPU Leeway以及Greylist Limit 等功能（什么鬼？）

升级

不过管它什么鬼呢，升级就是了，

编译完成后检查一下版本号：

nodeos --version

返回如下信息，版本号v1.8.5，看来没搞错。

替换原来的nodeos并重启，一切🆗！

CPU挖矿？

据说昨天EOS上发生了一件大神，那就是CPU炸了，好多正常操作没法进行。而这一切的罪魁祸首据说是XXX项目搞空投引起的。

XXX项目空投，你转任意金额过去，它会把相同的金额转回来，同时还会得到XXX账户内万分之一的XXX代币。而这个项目每秒挖25个XXX代币，5个代币留给项目方，20个留着空投，据说会持续好多个月。

也就是说你只要不停的转账，就能不停地撸到空投币，消耗的只有CPU资源，所以有人管这个叫做CPU挖矿。

据说有人做了所谓的矿机(自动转账工具)，然后开N个账户挖矿，难怪主网CPU炸了。

EOS行情

从非小号的数据上看，EOS最近30天的价格可谓波澜起伏。

国家支持区块链的东风让EOS也水涨船高了一波，可惜我在东风吹起之前都清的差不多了，悲催。
