(图源 ：pixabay)

天然冰场 & 打出溜滑

昨天早晨下起了小雪，到了中午因为气温半暖所以变成了雨夹雪。

到了晚上地面上积了厚厚的一层冰激凌一样的东西，路上特别泥泞，走在路上一步小心就会摔跤。就算不摔跤，出去走一圈，只要不是特别防水的鞋子，那么一定会湿的。

这还不算特别惨的，更惨的是，雨夹雪持续下了一整夜，然后早晨气温降到了零度以下，所以地面已经完全变成了冰面，好好的一个小区变成了天然冰场。

虽然物业人员清理了大半宿，但是能解决园区主路的冰面就不错啦，至于小路和小广场之类的就完全顾不过来啦。

有几个好朋友外出的时候摔跤啦，在群里哼哼唧唧呢，真是同情他们啊。

又得浮生半日闲（浪费了大半天的时间）

朋友夏天的时候骑摩托酒驾被抓，然后被罚1K+大洋，扣12分，驾照从A降级到B(具体A几B几我记不清楚了)，驾照吊销半年，学习七八次，还要参加考试。


(图源 ：pixabay)

据说马上就要到考试的截止期限了，并且考试的地方很远，于是前些天我和他说，哪天我陪你去考试吧。

今天早起天空飘起了小雪花，看天气预报，这几天有大暴雪又降温啥的，于是我给朋友打电话，趁着大暴雪还没来，赶紧去考试吧。

大概8:30多接到朋友，就开始赶赴考点，我们在城市的北边，考点在城市的南边，我从二环路走，差不多整整绕半个城市。

早晨8:30多正是早高峰，因为雨雪，还有两起车祸，虽然二环路都是高架桥，还是堵得不得了，蹭啊蹭啊，总算到达目的地了。

到达目的地发现没有停车的地方，于是我让朋友先下车，我继续沿着道路的方向慢慢溜达，溜达到一个大学的门口，正纠结要不要去大学内转转，然后接到朋友的电话。

朋友电话说：“到哪了，快回来吧，今天考点不能考试，要下周二才能考。”我擦。

接到朋友后我问他：“你咋不问明白就来了？”，朋友回答：“我上次来学习的时候问过啦，说可以随时来考，谁知道他们变卦啊。”


(图源 ：pixabay)

哈哈，好像这事还真怨不到朋友，于是我们又开车回来啦，大半天就这么废啦！

以讹传讹 & 推波助澜

前段时间一朋友在微信群里说迪卡侬丁香湖店撤店了，这真是让人万分遗憾，迪卡侬是我比较喜欢去闲逛的地方之一，看看各式各样的运动用品也挺好玩的。

作为迪卡侬的会员以及老用户，我也在它家买了不少用得上用不上的东西，比如跑步机啊、篮球啊、足球啊、跳绳啊、衣服鞋帽、滑板车等等。

所以听说迪卡侬撤店，心里还有些失落感呢，真希望是个假消息。

不过传出来这个消息的是我的一个朋友，她家就住在迪卡侬后边的高层里，下楼就是迪卡侬，想必得到肯定是第一手消息，应该不会错的。

知道了这个消息以后，有朋友问及或者在微信群里提及迪卡侬时，我也会告诉他们，迪卡侬丁香湖店撤店了，去别的迪卡侬店吧。

昨天下午外出办事，恰好经过迪卡侬，我想看看迪卡侬撤店后变成什么样子了。

结果一看，这不是正常营业中呢吗，而且营业的好好的，一点也没有关店或者撤店的迹象。

也就是说，我被朋友骗了，然后还把这个当成真事，到处宣传，那些又被我骗了的朋友，知道迪卡侬正常营业，该会作何感想呢？

当然了，我朋友没有骗我的动机和必要，她可能只是哪天看到迪卡侬调整东西摆放或者更新商品，就当成了撤店，问题是因为我对她的信任就完全当真了。

反省一下，其实不光这事，生活中有多少事，我们只是听说，就当真了呢？然后在经过我们传播出去，又会造成哪些不良影响呢？看来以后未经自己亲自核实的事情，还是不要乱传的好。😳

每天进步一点点：学习使用LVM

为了方便动态扩容，我这次准备使用LVM来管理空间，不直接在硬盘上分区、创建文件系统以及挂载了。


(图源 ：pixabay)

确定新硬盘

操作之前，先看用sudo fdisk -l 看一下我的新硬盘，可以看到新硬盘为/dev/sdc。

sudo fdisk -l

创建PV

首先要创建PV，使用的命令为pvcreate(Initialize physical volume(s) for use by LVM)，我直接在新硬盘上创建。

sudo pvcreate /dev/sdc

创建VG

创建VG使用的命令为vgcreate，这里我为VG起的名字为vg_storage。

sudo vgcreate vg_storage /dev/sdc

创建完成后可以用vgdisplay查看一下：

sudo vgdisplay

创建LV

接下来是创建LV，使用命令为lvcreate(Create a logical volume)。

创建线性的LV命令为：
lvcreate -L|--size Size[m|UNIT] VG

我使用了如下命令：

sudo lvcreate -n lv_main -L 3200M vg_storage

创建完成后，查看一下：

sudo lvdisplay

擦，我原本想创建的是3.2T，结果弄成了3.12G，囧。

扩容

既然创建错了，那就直接扩容一下吧，弄成3T吧：

sudo lvextend -L 3145728M /dev/vg_storage/lv_main

sudo lvdisplay

这次看起来没问题的

再来看看VG

sudo vgdisplay

可以看到总的PE以及已分配的PE

格式化& 挂载

接下来将lv格式化

sudo mkfs.ext4 /dev/vg_storage/lv_main

创建目录并测试挂载：

sudo mkdir /lv_main
sudo mount /dev/vg_storage/lv_main /lv_main

编辑fstab

sudo vi /etc/fstab

添加如下内容：

/dev/vg_storage/lv_main /lv_main ext4 defaults 0 0

重启，一切正常，搞定收工。
