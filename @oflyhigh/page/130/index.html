原来STEEM也可以做菠菜应用啊

要说在EOS上最火的应用是什么，答案无疑是菠菜应用了，各个项目方都快马加鞭的推出菠菜应用，玩家也是蜂拥而入，看来骨子里赌性都不小。


(图源 ：pexels.com)

我曾经思考过在STEEM上做类似掷色子游戏的可行性，但是考虑到STEEM没有智能合约，没有智能合约那么就需要第三方的服务程序和脚本，需要第三方的服务程序和脚本我觉得就有机会作弊，有机会作弊就觉得这样的菠菜游戏就会没人来玩。

我想了好久，没想到如何解决透明以及公平性问题，遂放弃。（不得不说，我放弃的各种想法太多了，一事无成的人都是这样，想的多，做的少😂）

前两天，@deanliu 刘美女推荐了一个Magic Dice游戏，这不就是我之前想过的掷色子吗？那么透明性以及公平性问题都解决了吗？

于是用了近乎小半天的时间去研究了一下这个掷色子游戏，不得不说，透明性和公平性问题解决的很巧妙，所以对于所有玩家而言，需要付出的只是提成问题，而不必担心有黑幕。

不像某彩，说是百分之XX给彩民，但是使用各种手段，最终真到彩民手里的微乎其微，有人用缴智商税来形容购买某彩的行为，倒是很是贴切。

尽管如此，我并不建议大家玩菠菜游戏，因为从总体/概率来看，庄家才是稳赚的。偶尔娱乐一下，倒是无所谓，博博运气嘛。


(图源 ：pexels.com)

其实我想表达的是，STEEM区块链应用是有无限可能的，只要你够聪明，设计出够巧妙的机制。其实相对智能合约而言，这个STEEM区块链上并未使用智能合约的菠菜游戏要更胜一筹呢，至少不用面对EOS上CPU和RAM等资源限制问题😀。

Updated my witness node to v0.20.8 / STEEM又双叒叕升级了

早晨瞄一眼STEEM的Github，神马，STEEM又双叒叕发新版本了，跟新说明如下：

This release fixes a fatal memory leak during reconstruction of the block log index. This update is optional so long as you do not need to reindex. However, if your block log index needs to be reconstructed, this update is required.

简单翻译过来就是，此版本修复了日志索引重建过程中致命的内存泄漏，如果你不需要重构块日志索引那么就可以选择更新或者不更新，否则就需要更新。

0.20.8 does not require reindexing from 0.20.7

如果是从0.20.7升级上来的，则不需要重建索引(重播/replay)

虽然说不需要重构块日志索引可以选择不更新，但是谁知道啥时候会需要呢，更新上来总是会更靠谱一些，那就更新呗。

老规矩，编译完成后看一下版本信息

steemd --version

执行也起来一切正常

尽管升级起来其实并不麻烦，但是这升级太频繁了点啦，但愿STEEM开发稳一点吧，毕竟这行情，见证人们也不愿意折腾呢😂

每天进步一点点：Excel的四舍五入 & 向下、向上截取

在我之前的帖子：原来Excel也很好用呀，提到我开始使用Excel完成一个以前一直用Word+计算器+手工填写来完成的一项周期性任务。用了一段时间后，发现还真是舒服啊，每次至少节省我半个小时以上时间。


(图源 ：pixabay)

但是还有有一点美中不足，表格中我计算出来几个数值，小数部分多达八位，我需要将这个数值填写到某一网站上去，长达八位的数值，填起来有点累，于是我就采取四舍五入的方式。

而所谓的四舍五入，就是我肉眼去看这个数字，然后在大脑里简单运算一下，四舍五入取小数点后四位。这样做一般来讲并没有什么问题，但是偶尔会遇到大脑短路，计算出来的结果跑偏；或者眼睛短路，肉眼看数字就看串位😂。

四舍五入

今天突然想，Excel这么高级的工具应该支持四舍五入吧？比如PHP和Python中都支持ROUND函数，那么Excel是不是也支持呢？

于是直接写了类似如下的测试

=ROUND(H32, 4)

其中H32是我之前计算出的一个数据对应的单元格，4表示保留小数点后4位，ROUND表示四舍五入。结果灰常完美，我猜对了，看来这些语言都有共同之处呢。

向上、向下截取

那么既然ROUND没问题，是不是CEIL以及FLOOR都可以用，并且行为和PHP啥的都一致呢？测试了一下，原来Excel中这两个函数分别叫做CEILING与FLOOR，并且比其它语言中的CEIL以及FLOOR高级多了。

语法如下：

CEILING(number, significance)
FLOOR(number, significance)

分别代表返回将参数 number 向上/下舍入（沿绝对值增大/减少的方向）为最接近的指定基数的倍数，是不是很有意思？

当然了，对我而言，这两个函数一般我用不到，Excel还有两个弱化版的处理函数，分别为：

ROUNDUP(number,num_digits)
ROUNDDOWN(number,num_digits)

不同于四舍五入，这两个函数分别代表向上截取，和向下截取，支持小数，虽然是弱化版，但是也相当强大了。


(图源 ：pixabay)

总之，越来越喜欢Excel啦。

开始Power UP 100% & 如何设置

在以往一个常常被讨论的话题就是Power UP 100%以及Default: 50% SBD / 50% SP如何选择更合算，我曾经在不同时期写过几个帖子和大家探讨这个问题。这个问题判断的主要依据就是喂价以及内部市场价孰高孰低。


(图源 ：pixabay)

但是最近虚拟币市场的暴跌导致STEEM&SBD也跟着暴跌，并且因为SBD的脱锚，喂价机制已经形同虚设了，所以喂价大幅高于/低于内部市场价的情况，将很难出现了。所以对于不想将收益马上变现的人而言，如何选择差异并不大。

最近Power UP了几次，都是先把零星的SBD在内部市场换成STEEM，然后在把所有的Power UP，相当麻烦，所以直接设置成Power UP 100%了，这样省心省力。

Power UP 100%有两种设置方式，一种是发帖时临时设置，只针对当前帖子；一种是全局设置，针对这个账户的所有帖子或者回复。

临时设置

临时设置，只需在发帖界面底下，点击Advanced settings

选择你倾向的方式，并保存即可。

全局设置

首先进入Settings

然后在Preferences中选择你倾向的设置即可(其中上边的针对主贴，下边的针对回复)

是不是非常简单，不想随时变现又懒得经常Power UP的小伙伴们可以试试哦。

曾经以为自己是专家，原来不过是小虾

17年币圈整体都是大牛市的行情，用以往常说的一句话：“站在风口上，猪都能起飞。”我误打误撞，不小心混到币圈来，也买了一些币，眼看着几倍几十倍的上涨，我不禁有些膨胀了，觉得自己简直就是个投资专家。


(图源 ：pexels.com)

每天看着财富不断增值，要说心如止水那是骗人的，其实还是蛮激动的。激动来激动去，觉得自己还是蛮有眼光的嘛，你看虽然有几个币被套，但是我也投资了好几个10倍币甚至百倍币，说自己不是专家，自己都不相信。

因为平时一副专家范，再加上各路媒体都大肆宣传、介绍比特币等虚拟货币，所以也有一些朋友找我推荐靠谱的虚拟币。虽然我很膨胀，但是不傻，推荐虚拟币这种操作可不容易，万一推荐错了，会挨骂的。

但是碍不住朋友们苦苦哀求，我只好按我自己的了解介绍了几个我认为靠谱的币，我觉得即便不涨它个十倍百倍，但是至少应该不会亏钱吧。不过我还是声明了虚拟币有风险投资须谨慎，归零也不是不可能的，我的介绍仅供参考，不作为投资建议。

但是想必他们没把我的话听进去，而是当了耳旁风，然后我推荐的、认为不会亏钱的币，跌掉了96%，然后，我这几个朋友现在都不理我了。这充分说明我的朋友还是不错的，至少没来砍我，也没来骂我……

不过这并没有打击我作为专家的信心，大势所趋嘛，跌90%以上的虚拟币比比皆是，并非我推荐的东西不可靠。作为专家我应该寻找投资机会，在别人恐惧时贪婪，继续加仓。

然后，前些天，我又投了一些，然后今天早晨一看，有的腰斩，有的归零……


我躺在床上，这样拿着手机，想了半天之后，才想明白，原来自己根本不是什么专家，只不过是一条随波逐流的小虾罢了……

就像2008年股市的大牛市，我在6000点还一路买买买，然后用至今仍在套牢的事实证明了自己不是股神；就像300多元开始买黄金，一路买到现在，还没回本，证明自己不是贵金属交易天才。

总得要付出一点代价才会看清自己，才能避免继续膨胀下去——就是这代价有点大了😭

见证人收益大幅缩水

大约两个月以前，我曾经写了一篇文章：《揭秘见证人系列：见证人收益几何？》，里边介绍了见证人收益的来源以及大致的见证人收益情况。


(图源 ：pixabay)

两个多月过去了，见证人收益有什么大的变化了吗？其实从STEEM的经济体系来看，以STEEM作为基础货币来衡量的话，见证人收益几乎没啥变化。

那么我是纯粹的标题党吗？也不尽然，实在是STEEM对法币跌的太多了😂

以两个月前TOP20见证人为例，当时他们每天出块奖励约为283.61 STEEM，按当时约5元人民币/STEEM的，月收入约42500元😍，什么服务器成本啥的都不在话下。

我当是每天出块奖励约为3.88 个STEEM，每月约582元，虽然还远远不够支付服务器费用，但是如果不考虑成本，582元月收入也不错了😂，至少每天换一个汉堡+可乐还是够的。

然而，现在TOP20见证人的奖励几乎没变，当天每天约302.55个STEEM，不过STEEM价格跌到了1.65元左右，算下来每月收益鱼约为15000元，基本上还可以维持下去。

得益于见证人得票的增加，我现在每天见证人收益约为6.17个STEEM，折算成人民币每天约10.18，每月约305元，这价钱每天换不来汉堡+可乐了，只能刀削面+老雪花啤酒了。

如果再想到我的服务器成本，有点想哭了，我在慎重考虑似乎可以关掉备份节点服务器了，省点是点，毕竟这行情挥霍不起啊，另外就是STEEM程序现在很稳定，貌似有无备份节点区别不大。

如果你身边有做STEEM见证人的朋友，并且还不是TOP20，那你一定要对他好一点，没事请他吃吃饭，多陪他喝喝酒，毕竟他太可怜了😭
