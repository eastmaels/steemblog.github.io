刷脸支付 & 靠脸吃饭

总看各路媒体上说刷脸支付刷脸支付，但是连手机刷脸解锁都觉得特傻的我，觉得刷脸支付一定是傻得不得了，不过昨天去超市还真的体验了一把刷脸支付。


(图源 ：pexels.com)

话说昨天去家乐福买了一些零零碎碎的东西，结账的时候发现每个结账的通道都人好多，这时突然发现最左侧有自助结账区，就走了进去。

说是自助结账，也是也有服务员在一旁看着指引。自助结账区分为两侧，分别是支付宝和微信，看来尽管银联支付之类的奋起直追，但是要达到支付宝和微信的流行度还是任重道远啊。

微信那端人太多，我选了支付宝这边的一个机器，在服务员的指引下扫描了所有商品，然后进入支付界面，发现有一项是刷脸支付，果断试试。

先是按提示开通，然后把一张大脸凑过去，然后滴，支付成功！整个过程非常刘畅，这还是我首次开通的缘故，估计以后再支付会更简单吧。

我在想，以后我上街要不要弄个纱巾把脸蒙上？不然万一被人偷偷地刷脸支付了，岂不是惨了？没想到这张脸突然也可以变得很值钱，以后餐厅之类的全支持刷脸支付，我岂不是就可以靠脸吃饭了？


(图源 ：pexels.com)

想起一个段子：

曾经我也是靠脸吃饭的，直到后来差点饿死，然后才开始努力搬砖了。。。

如何将PEM私钥转换成PPk私钥

Amazon EC2创建实例时需要关联Key Pairs，之后可以用Key Pairs以及默认用户登录。


(图源 ：pexels.com)

但是我在Putty中指定从Amazon下载的PEM文件并尝试登录主机时，却提示我无法使用

Unable to use key file “C:\Users\oflyhigh\Downloads\TestEC2.pem” (OpenSSH SSH-2 private key (old PEM format))

那么怎样才能用Putty登录以及对应的私钥登录EC2的实例呢？原来在登陆之前，需要对私钥进行一下格式转换。

转换私钥格式使用的工具为：puttygen.exe，可以在Putty网站下载。最新版本的下载地址可以在此处找到：
https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html

操作步骤如下：

打开puttygen.exe，点击Load按钮：

选择对应的PEM文件，puttygen.exe会弹出如下提示：

关掉提示，在箭头指向处填入密码，并在下方填入确认密码

选择Save the private key保存私钥

保存的私钥格式(扩展名为).ppk，就可以在Putty中使用啦
