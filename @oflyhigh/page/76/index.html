商业机密

我的一个亲戚家经营一家连锁药店，生意做得很大很红火，是我们这帮穷亲戚中的有钱人。😀


(图源 ：pexels.com)

这两天见到了这个亲戚一次，她和我抱怨现在生意越来越难做了，各个方面经常搞检查非常严格不说，同行之间的竞争也很激烈。

她给我讲她的管理药店之前用的一款软件功能不强大、操作也很不方便，所以一直有换一款软件的想法。

她的一个同行知道这件事以后，极力推荐自己家正在使用的一款软件，说是功能强大、操作方便用于管理连锁药店非常方便。

我的亲戚考察过这款软件后，发现属实不错，但是为了安全还是没从同行推荐的渠道那里购买软件，而是过了半年以后通过其它渠道购买。

换上软件后，发现软件属实让人很满意，想要的功能啥都有，不过这生意怎么越来越惨淡了呢？让人百思不得其解。

后来我这个亲戚的一个朋友的朋友在亲戚同行的药店工作，透漏给我的亲戚：“你还蒙在鼓里了啊？你的药店的底裤都被人扒光了，什么购药渠道、成本价、卖价等信息，同行那全都知道。”

亲戚陷入巨大的震惊和愤怒中，想也不用想，就知道是管理软件的问题，没想到千防万防，还是被人利用了。不过这玩意愤怒也没用，抓不到实质的证据。

亲戚换掉所有的软件，又经过好长一段时间的打拼，生意才重新渐渐红火起来，不过这次打击，也算是损失惨重了，况且以后可能还有一些不良影响持续很久。


(图源 ：pexels.com)

我听了后也是相当震惊，窃取商业机密啥的以往觉得是电视剧里的剧情，没想到真真切切发生在亲戚身上。

当然了她所谓的朋友的朋友能把这个信息反馈给她，是否算是她在对方那安插的钉子，就不得而知了。😳

O哥闲扯淡：或许燃烧SBD/STEEM，才是STEEM区块链的最佳出路

大家都知道HF21/HF22带来许多改进，包括SPS以及EIP等等，而EIP包括独立的点烂池(downvote pool)、50%的curator收益、5分钟的早鸟惩罚时间以及收益曲线调整等等。


(Image source: https://pixabay.com/)

但是EIP引进的变化会给STEEM带来足够想象空间或者说能拉升币价吗？我看未必，EIP带来的不过是新一轮的博弈，不过是财富分配上的一次调整，但是不能从根源上解决币价不景气的问题。

币圈流行一句话：拉盘才是硬道理，但是对STEEM而言，如何才能拉盘呢？

STEEM是一个通胀的系统

我们知道，STEEM是一个通胀的系统：持股(SP)者分红、见证人工资、作者/Curator收益，都是从通胀的币中分配，HF21/22之后，又加了一项SPS的提成。

但是这些分配并没有消耗STEEM的供应量，也就是说系统里/市场上，STEEM/SBD会变得越来越多，抛压增加，那么币价很难拉升。

另外还有一点，STEEM系统中没有手续费一说，包括转账、发帖在内的任何操作都不消耗手续费，这也导致STEEM/SBD没有减少的机会。

有朋友可能会问，买赞不是要花费STEEM/SBD费用吗？其实买赞/卖赞消耗的STEEM/SBD，只不过是资产在两个用户之间的转移。

还有朋友问，点烂（Downvote）不是减少了作者的发文收益吗？这个应该能在一定程度上降低通胀率吧？答案是并没有，点烂减掉的收益相当于重新回归奖励池，简单来讲就是，你得不到就会给其它人。

promote功能与@null账户

那么STEEM中有没有燃烧（消耗）SBD/STEEM的方法呢？这样就可以降低通胀，减少总量，进而提升币价呢？答案就是promote功能。

‘promote`功能，简单来讲，就是通过转账给@null账户，来在promoted栏目下，提升排名的手段。转给@null账户的STEEM/SBD，会被系统销毁。

这是一个很好的功能，既能宣传帖子，又能消耗SBD/STEEM，一举多得。可是因为promoted栏目并没有人看，这个功能几近于白费。

降低系统通胀的方法/拉盘的方法

那么有没有可能充分利用这个功能呢？或者说如何利用@null账户的功能来降低系统通胀呢？

我想到的可能有两点：

  • 将promoted栏目合并到Trending栏目，也就是说按照一定的规则和权重计算，promoted花费越多的帖子，会获得越高的Trending排名。

  • 点烂（Downvote）减掉的钱，并不回归奖励池，而是发送给@null账户。

第一点，改前端就可以实现，效果应该是立竿见影；第二点应该需要硬分叉，不过看起来应该是百利而无一害。

结束语

扯淡神马的，我最擅长了。

至于系统该如何改进，STEEM将来如何走，就不是我能把舵的啦，闪喽。

升级本地EOS节点至v1.8.3

今天注意到EOSIO在github上释出了新版本EOSIO v1.8.3，话说这个版本推出间隔有些短，主要是因为v1.8.2中存在BUG，导致异常的高带宽占用。

不幸的是，前段时间手欠把本地EOS节点升级到了v1.8.2 确实也遇到了高带宽占用这个问题，那么只好升级喽。

升级EOS

所幸编译这些东西已经轻车熟路啦，卡卡卡一通命令执行下来即可，至于编译程序用多长时间，那又有什么关系，耐心等待就好喽。

编译完成后检查一下版本号：

nodeos --version

返回如下信息，版本号v1.8.3，看来没搞错。

弃用network-version-match

v1.8.3 除了解决一堆bandwidth和p2p(Net plugin)的BUG外，最重要一个修改就是弃用了network-version-match选项。

这个限制方向很有趣，原本我以为是p2p限制其它不同版本的节点链接，后来才搞清楚是限制节点连接不同版本号的P2P节点。

不过现在不用晕啦，这个选项已经被弃用了。

V2.0.0 版本？

在EOSIO v1.8.3 Release Notes，讲述弃用network-version-match选项时提到在v2.0.0版本：

Setting this option to false is especially important given that the v2.0.0 release of EOSIO will add a new supported version of the network protocol that enables additional capabilities…

看来下个大版本有可能就不是v1.9.x了，而是直接v2.0.0，另外，这个大版本已经会引入更多的新功能和新特色，好期待。

共识升级

另外一点就是，距离1.8共识升级，还有不到10天啦。详情参考：1.8: Consensus Protocol Upgrade Process吧。

汽车住院了

我的汽车虽然不是什么豪车，但是一直很给力，小毛病很少。尽管有一次我不小心开到路边的工程车上撞报废一个车门，还有忘记关灯废掉一个电瓶，但是这都是我使用而非汽车本身的问题。


(图源 ：pexels.com)

不过今天汽车出毛病了，跑了几个小时的高速，下高速到收费站交费（我没有安装ETC），交费完毕再按关窗按钮，我听到咣当咣当两声。

我以为是我撞到收费站的栏杆，或者被后车追尾了呢，然后开出一段距离后把车停到路边，下车检查，结果没有发现任何问题和碰撞的痕迹。

那是什么声音呢？我带着疑问重新启动并前行，发现开车也没有任何问题。于是就想把车窗关上继续前行，结果发现车窗死活关不上了。

侧耳倾听，按车窗按钮时能停到电机工作的声音，但是车窗一动不动。于是我尝试用手把车窗按下来，成功地按下来之后，发现再也上不去了。

这下惨了，要是晚上下雨，岂不是要灌一车水？没办法，只好联系我开车会馆的朋友。

朋友听了我的描述后，告诉我可能是车窗升降器里的钢丝断了，需要拆开看看。只好把汽车开到朋友的车会馆那，让他去处理喽。


(图源 ：pexels.com)

不过还好是我下高速公路的时候它才出毛病，如果是上高速的时候出毛病，车窗关不上，我岂不是要在高速公路上吹风了？

那滋味想想就无比酸爽，哎。😔

QQ绘图红包

最近一些年，由于微信的流行和冲击，QQ是越来越没落了，除了一些中小学生在用，就是一些有特定主题的群体了，比如说程序员，因为在QQ群里分享一些代码片段还是很方便的。

不过QQ一直没有放弃进步，比如前段时间推出的SVIP9会员，又比如一直以来被QQ玩出各种花样的红包。

什么口令红包就不用说啦，我一直以为这个口令红包相当扰民，并且经常被一些SPAMER利用，用来发一些垃圾广告。

语音红包更是变态，弄个红鲤鱼绿鲤鱼或者黑化肥挥发会发灰，灰化肥挥发会发黑，读起来折磨死人。

不过今天又被QQ红包的一个新功能刷屏了，那就是绘图红包，早晨刷一个常混的QQ群，我才发现我的朋友当中有这么多被编程耽误的艺术家啊。

看了上边这些灵魂画手的作品，我不禁感慨，我这些朋友一定是程序员群体中最好的画家吧？

对了，封面图的钢琴是我画的，我觉得我也可以考虑改行啦！

雨中双泉寺

小时候学李健吾的《雨中登泰山》的时候，我心想虽然作者写得挺好挺美，但是更多的应该是无奈吧。

毕竟如果可以在天气好的时候登山，作者也不会选择冒雨上山。原文中的这几句，道尽了作者的无奈：

偏偏天公不作美，下起雨来，淅淅沥沥，不像落在地上，倒像落在心里。天是灰的，心是沉的。

我其实冒雨去双泉寺，也很无奈，毕竟出发时候虽然天气有些阴沉，但是看着不像马上有雨的样子。

没想到开车到半路，淅淅沥沥下起了小雨，到了双泉寺，更是大雨如注，所以我慢慢逛仔细欣赏双泉寺美景的美好愿望落空了。

尽管雨下得很大，但是还是有虔诚的香客冒雨上香。

雨中双泉寺也别有一番风景，但是身上很快淋得通透，所以匆匆忙忙拍了几张照片，便原路返回了。

回程路上，大雨更加疾骤，我庆幸没有在双泉寺更多停留，否则该变成落汤鸡了吧。
