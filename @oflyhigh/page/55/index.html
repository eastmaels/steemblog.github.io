升级本地EOS节点至v1.8.5 & CPU挖矿？

Blockone在15天以前发布了EOSIO v1.8.5，据说解决了Net plugin的一些问题，并且加入Subjective CPU Leeway以及Greylist Limit 等功能（什么鬼？）

升级

不过管它什么鬼呢，升级就是了，

编译完成后检查一下版本号：

nodeos --version

返回如下信息，版本号v1.8.5，看来没搞错。

替换原来的nodeos并重启，一切🆗！

CPU挖矿？

据说昨天EOS上发生了一件大神，那就是CPU炸了，好多正常操作没法进行。而这一切的罪魁祸首据说是XXX项目搞空投引起的。

XXX项目空投，你转任意金额过去，它会把相同的金额转回来，同时还会得到XXX账户内万分之一的XXX代币。而这个项目每秒挖25个XXX代币，5个代币留给项目方，20个留着空投，据说会持续好多个月。

也就是说你只要不停的转账，就能不停地撸到空投币，消耗的只有CPU资源，所以有人管这个叫做CPU挖矿。

据说有人做了所谓的矿机(自动转账工具)，然后开N个账户挖矿，难怪主网CPU炸了。

EOS行情

从非小号的数据上看，EOS最近30天的价格可谓波澜起伏。

国家支持区块链的东风让EOS也水涨船高了一波，可惜我在东风吹起之前都清的差不多了，悲催。

传销洗白白？

之前没少说过阻止朋友陷入资金盘或者区块链传销陷阱，不过如果有一个朋友陷入资金盘陷阱很深，并且他知道是资金盘，那该怎么办呢？


(图源 ：pixabay)

今天一个朋友突然打电话过来，问我央行数字货币的事情，咨询我到哪里可用买到矿机挖央行的数字货币。💀

好吧，恕我孤陋寡闻，我还真不知道如何挖央行的数字货币，或许努力上班工作就是挖矿的正确方式？如果能用矿机挖，大家都不用上班了，都买矿机在家赚钱就好了，想想那场面也是醉了。

之后又和朋友随便聊了些其它事情，他告诉我他购买了N处房产，还持有N多股票，还买了许多实物投资金条以及好多纸黄金。

我夸朋友真是成功人士啊，朋友谦虚地说，成功啥啊，投的P2P平台暴雷跑路，亏了好多，真点背啊。

然后我又开始安慰他，告诉他财去人安乐等等，虽然P2P损失一些钱，但是在其它方向上投资还算成功云云。

结果说着说着，朋友又开心起来，他说因为P2P暴雷，他加入了这个P2P平台的维权群，然后维权群里有人搞什么购物返利平台，他跟着参与，又在群里发展了几个下线，现在收益还不错呢。

我擦，这不是传销嘛？我告诫朋友，这些都是传销，千万不要沉浸进去。朋友回答没事的，他知道是传销，现在已经是利润在玩了。

好吧，利润在玩，那肯定是不会亏钱了。不过传销你赚钱，不就意味着你的下线赔钱嘛？如果这个传销盘崩盘，那么你的下线或者下线的下线会不会来找你维权呢？有没有可能被抓进去呢？

朋友听了以后，品味了一会，发现还真的有这种风险，于是告诉我他决定果断撤出这个传销游戏了，并且准备挨个通知下家，他觉得这个东西有风险，决定不玩了。

过了半天以后，他又给我打电话，和我说他已经把下家通知了遍，自己也决定不玩了。哈哈，这算是传销洗白白吗？如果这个传销崩盘，他算是受害者还是会被抓进去呢？


(图源 ：pixabay)

总之，传销/资金盘还是不要沾，亏钱了心疼自不必说，就算是赚钱了，也不安稳啊。

每天进步一点点：Linux下校验文件

在之前的帖子中，我介绍了axel这个多线程下载工具，用来替换wget进行多线程下载。

每天进步一点点

(图源 ：pixabay)

不过在用axel下载了一个数百G的文件后，我突然发觉心里没底，万一这个文件里边丢失哪怕一个字节的内容，这个文件都算白下了，那么如何校验我下载的文件和原始文件一致呢？

md5sum

其实最简单的方法就是校验文件的hash值了，而校验文件hash值最常用的就是md5sum啦。

Ubuntu 18.04发行版中已经包含了md5sum（包含在GNU Core Utilities中），所以我们直接使用即可，无需安装。

调用方式如下：

md5sum [option]… [file]…

比如我要检查文件aaa.tgz的md5，那么只需调用：

md5sum aaa.tgz

输出结果如下：

之后再去服务器上执行同样的命令，获取服务器对应文件的hash，并进行对比就可以啦。

使用校验文件

在对单个文件进行处理时，只要不是头昏眼花，上述人工肉眼判断是没有问题的，但是如果多个文件需要校验，那么估计就该头昏眼花了。

所以使用-c参数就很有必要了：

-c, --check read MD5 sums from the FILEs and check them

简单的来讲，就是把原始文件生成的md5sum输出写入到文件中，下载或复制等传播后，md5sum会直接处理文件并对比校验码。

举例如下：
我们在服务器上生成aaa.tgz的校验码并保存到本地文件md5.txt，内容如下：

d41d8cd98f00b204e9800998ecf8427e aaa.tgz

在本地执行如下命令：

md5sum -c md5.txt

返回内容如下，说明文件校验成功。

下面我们手工破坏一下文件，再测试一下：

echo "hello" >> aaa.tgz
md5sum -c md5.txt

纸上得来终觉浅：lxc初体验

最近尝试去了解一下Linux下的虚拟机以及容器之类的内容，知道了lxc、lxd这些神奇的东西，lxc是Linux Container的缩写，简单的来讲在内核的基础上进行轻量级虚拟化，以便隔离进程和资源。


(图源 ：pixabay)

lxd是lxc的进化版，先试试lxc吧。

安装 lxc

先来安装一下试试：

sudo apt install lxc

从提示信息可见，它还一并安装了bridge-utils libpam-cgfs lxc-utils这三个东东

安装 lxc-templates

安装之后查看一下模板信息：

ls /usr/share/lxc/templates/

发现只有这么四个东东，没有Ubuntu啊

于是再安装lxc-templates吧

sudo apt install lxc-templates

好嘛，这就是传说中的全家桶嘛？

装完之后模板果然多了好多

创建Container

接下来使用如下命令创建容器：

sudo lxc-create -t ubuntu -n xxx

接下来大量的信息以及漫长的等待，这个过程更有点像网络安装ubuntu系统：

快完成后有这样两句输出，这就相当于烧写磁盘啦

Copy /var/cache/lxc/bionic/rootfs-amd64 to /var/lib/lxc/xxx/rootfs ...
Copying rootfs to /var/lib/lxc/xxx/rootfs ...

最后这段写着初始用户名和密码，也很重要，之后登录要用到这个

查看Container

我们可以使用如下命令查看我们已经创建的lxc 容器。

sudo lxc-ls

进入Container

我们可以使用如下命令进入到Container界面

sudo lxc-console -n xxx

这两个特别的指令要记一下：

登录成功后发现和正常安装的Ubuntu没啥区别

其它

使用uname -a可以看到内核版本与主机的版本是一样的。

使用lscpu可以看到CPU信息和主机是完全一致的。

如果想使用ifconfig查看网络信息则需要先安装net-tools

sudo apt install net-tools

总之，看起来也挺好玩的，不过据说现在已经没谁玩啦，哈哈。

收获了一个点赞

开车出去办事，在经过一条马路时，一个约70岁的大妈从道路右侧直接横穿马路冲了过来，貌似很着急的样子。


(图源 ：pixabay)

如果我继续保持车速前行，那么要么逼迫大妈退让，要么会撞上大妈。但是无论哪种选择都不太好，于是我主动放缓车速，等大妈通过。

原本以为等大妈通过就算完结，我继续赶我的路，结果大妈突然在我的车前方两三米处停住，吓得我赶紧刹车，大妈这是闹那样，要碰瓷吗？我极其忐忑。

还好我担心的事情并没有发生，大妈并没有倒在地上讹我，而是伸出右手，对我竖起了大拇指，给我点了一个赞👍。

这真是让我始料不及，没想到我稍微避让一下大妈，竟然让大妈特意停在危险的马路上给我一个赞。

不过大妈你这样做真的很危险，你知道吗？首先过马路一定要走人行横道并且看红绿灯，横穿马路是非常危险的，路口就在20米左右的后方，多走几步道就当锻炼身体了。

另外，既然决定横穿马路，那么就好好的横穿马路，你这突然停下来，不但让我胆颤心惊，你也很危险，万一我刹不住车呢？

第三，你停下来给我点个赞，耽误我的时间也无所谓，可是后边一排车都要等着，这个赞收到的实在是尴尬。


(图源 ：pixabay)

虽然收到一个点赞有些小感动，但是还是希望大妈大爷们不要横穿马路，毕竟安全第一呀。
