一不小心将虚拟币转移到老旧的地址

大家都知道，交易所间的虚拟币转账一般都需要一个地址加上一个MEMO，地址一般是交易所对应币种的热钱包账户，而MEMO是用来区别交易所内部不同用户的，所以两项信息都要正确。


(图源 ：pixabay)

为了避免把这些信息搞错，我一般都把对应币种的交易所充值信息保存到记事本里，需要用的时候复制粘贴，这样大大的降低了出错的概率，这些年我一直这样用，一直没出过错误。

不过前些天我转了两笔虚拟币到某交易所，还是这样操作，就遇到了一个大麻烦，价值几万的虚拟币不知道卡到哪里，眼看着币价嗷嗷跌，不知道啥时候才能到账。

话说，前些天，当我间隔一天转完两笔之后，就去交易所打算挂单，然而左等右等还是没看到虚拟币入账，而这时距离第一笔转账已经过去了72小时以上，我基本可以确定，一定是出问题了。

问题出在哪里呢？我去账户钱包里看了一下，才发现对应币种的充值按钮旁边，多了个感叹号，点开之后发现提示说钱包账户已经修改，需要重新生成地址。

再看一下上述信息的提示时间，恰好发生在我第一笔转账的前一天，这就意味着，我之前的两笔转账，都转到旧的地址当中去了。

擦，这下我可傻眼了，两笔转账的虚拟币合计价值数万元，这要是转到黑洞里，我就亏大了。为什么转账之前没确认一下充值地址呢？为什么这个交易所没提前发送邮件通知呢？哎。

不过事情业已发生，懊悔是无用的，只好尝试联系交易所客服喽。将这个情况说明后，对方说会尽快处理，然后就没了下文，只好耐心等待啦。


(图源 ：pixabay)

以后转账之前还是要确认一下，不然真的是太危险了！但愿交易所不会不认账吧，想想真是郁闷啊。

简易火锅

前天的朋友买的老杨家熟食喊我喝酒，但是只喝了两口酒，吃了两块鸡架，程序的报警器响了，于是只好放弃酒局回家处理，十分失败。


(图源 ：pixabay)

朋友们为了安抚我前天受到的伤害，昨天白天决定晚上请我吃火锅，喝点酒，败败火，哈哈。

不过考虑到火锅店的牛羊肉有可能是加工的拼接肉，所以决定我们自己买肉，自己在朋友的会所涮火锅。

其中一个朋友去买了整块的牛羊肉，让老板冻上后，再给刨成牛羊肉卷，这肉就相当靠谱，比去外边火锅店安全美味的多。

又有的朋友负责准备海鲜，又有的朋友负责准备青菜，还有的负责洗菜，还有的负责刷碗，还有的负责去选购调料，当然了，还有我——负责吃。

从楼上办公室搬下来两张办公桌，然后就可以开吃啦。其实我去的比较晚啦，到那以后发现什么都准备妥妥地，所以除了吃喝，我也帮不上啥忙了。

据说火锅底料是我们中的一个朋友从成都出差特意带回来的，涮几片肉吃，果然是麻辣鲜香啊。

我们这次火锅一共来了八九个朋友，有会所的老总、有大学的校长、还有另外一所大学的教授、还有一个小学的音乐教师、一个博物馆的馆长、一个车企的高管、一个汽车配件的供货商、一个运营商的区域负责人。

虽然没凑齐三教九流，但是也是各行各业的都有啦，大家一起吃吃火锅喝喝酒，再看看不知道哪队和哪队（我外行）的篮球比赛，所有的烦恼都抛到九霄云外啦。


(图源 ：pixabay)

有朋友，有酒，有肉吃，真好。

言而无信

最近被人鼓动想投资一个二手的小房子，26平多一点，要价55万，在我们这个房价不算太高的城市，单价已经突破天际了。


(图源 ：pixabay)

看过房子后，其实并不是十分满意，因为楼板上看有被冲水泡过的痕迹，问了一下中介，据说是去年楼上地热爆裂导致，看着薄皮的屋顶，还是有些闹心的。

不满意的结果就是砍价喽，其实我觉得这个房子多说值52万，不过中介问过后说房主很牛，不差钱，多说给优惠5000，然后因为房主和中介认识，关系很好可以额外再优惠2000。

尽管这个价格超出我的预期很多，但是二手房嘛，哪有价格让双方都满意的，另外最近股票也赔掉不少，虚拟币也跌掉不少，所以想着算了，不计较那么三两万块钱的，54.3W就54.3吧。

于是今天让中介和房东敲定一下，有没有其它问题，没啥问题的话，我这边准备钱，然后三两天内过去签约。

中介问过房主后，告诉我没有任何问题，让我安心准备钱，然后到位了通知她一声就可以。

于是上午我这边开始把一些理财产品、货币基金什么的都开始赎回，然后合计再割肉一些股票，凑凑应该可以差不多了。

结果下午2点多钟，中介给我发来消息，房主反悔啦，房子又决定不卖啦！这把我闪的。

其实这房子原本我也不是非买不可，况且我们协定的价格其实还是偏高的，房主这价格卖出至少在当前来看，肯定是不亏的。

然后就是我看这个房子和准备购买这个房子，也投入不少精力。最关键的是，今天上午都敲定好了，然后我这边理财、基金什么都开始赎回了，他给我玩这么一手。

当然了，这事也怪我上午懒得跑去一趟交个定金，签个协议啥的。但是一则想着跑来跑去太麻烦，二则合计每个人都和我一样信守承诺呢。

其实若是这期间有人出高价买走了这个房子，这都无所谓，因为没签合同，当然是价高者得了。但是明明敲定好的事情，房主言而无信反悔不卖，这就让人恶心了。


(图源 ：pixabay)

哎，算了，反正这个房子又贵又不是非常满意，不卖就不卖吧，尽管着实被恶心了一下。

失败的酒局

朋友打电话告诉我，他买了许多老杨家熟食，让我一起去喝一会，不过考虑到晚上有很多事情要处理，加上戒酒，就果断拒绝。


(图源 ：pixabay)

结果朋友再三劝说，哎，我这人心软，于是就是收拾收拾去朋友那里啦。到了一看果然好多熟食，鸡架、鸡爪、鸡头、干豆腐、鸡翅，都是我爱吃的。

除了熟食还有桔子、苹果、花生、瓜子，当然了，酒也是必不可少的，有雪花、龙山泉以及梅河三种品种，怎么感觉像新年茶话会呢。

我们三人又在微信群里喊了几句，可是这次没有喊来其它朋友，另外一个逢局必到朋友，据说最近家教比较严，媳妇说了再出来喝酒就 打死他，所以这次没喊出来。

于是只好我们三人开喝喽，掀开一罐龙山泉，吃几块鸡架，虽然晚餐我已经吃得很饱，但是还是觉得酒很爽口，菜很美味。

不过刚喝两分钟中不到，家人打电话过来，说我设置的一个报警器报警啦，擦，这说明我一个很重要的程序出了故障，哎，早不出故障晚不出故障，单单等我这边酒瘾勾上来了才出故障，简直气死人啦。

不过好在我是有备而来了呢，带着IPad呢，打开手机热点，用IPad连上去，然后登录SSH查看到底哪里出了故障，结果我擦，连不上去。

无奈只好再喝一口酒，吃一口菜，跑路回家了。朋友说等我回来，不过我忙完已经到晚上10点啦，也懒得出去啦。


(图源 ：pixabay)

哎，不但破了戒酒的成果，而且没喝几口酒，没吃几口菜！第一次喝酒这么失败呀。

使用smartctl 来检测硬盘信息/通电时间/错误状态等

之前的帖子中说到从淘宝买了两块希捷16T的企业级硬盘，但是总觉得有点不放心，不过关于外观以及PCBA版本号的问题，淘宝客服回复说是工艺问题，具体的就一问三不知了。


(图源 ：pixabay)

要说就此评定这硬盘有猫腻，有点过于武断了，况且希捷官网上还是可以查询到保修信息的，那么把硬盘装机器上试试吧。

hdparm

装到机器上后，最基本的操作就是看看硬盘读出的序列号和盘体序列号是否相符了。最简单便捷的操作就是使用hdparm了

sudo hdparm -i /dev/sda

至于返回信息其实我也读不懂，不过好在序列号和盘体上标识的一致：

其中-i选项的含义如下：

-i Display drive identification

如果想显示更详细的信息可以用-I选项，含义如下：

-I Detailed/current information directly from drive

更多详细内容可以用hdparm -h指令获取帮助。

smartctl

看完序列号后，我想看硬盘的一些更细节的信息，比如上电次数，上电时间等等。这时候好像hdparm就捉襟见肘了（也可能是我没会用），不过好在我还知道一个smartctl 。

如果机器上没有安装smartctl相关软件，那么需要使用如下指令安装：

sudo apt install smartmontools

安装完成后，可以查看硬盘信息啦：

sudo smartctl --all /dev/sda

返回的信息很多，比如如下的基本信息：

SMART一些相关信息

我关心的一些详细信息：

其中我较为关心的有以下两点：

4 Start_Stop_Count 0x0032 100 100 020 Old_age Always - 1
9 Power_On_Hours 0x0032 100 100 000 Old_age Always - 0

也就是说上电次数为1次，通电时间0，这个结果说明硬盘是全新的啦。（有没有造假手段我就不清楚啦）

除了查看序列号、通电时常等信息外，还可以用smartctl对硬盘进行测试，可选的测试类型有如下这些：

比如：

sudo smartctl --test=short /dev/sda

进行不同种类的测试需要长短不一的时间，测试完成后可以用如下指令看结果：

sudo smartctl --all /dev/sda
