每天进步一点点: Favicon

大家访问网站时，有的网站会在浏览器标签栏上显示一个小小图标，有的则没有，这是什么黑科技呢？比如STEEMIT的网站上，就会显示一个绿色的小鸡图标（哈哈，是小鸡不？）


(图源 ：pixabay)

其实这个秘密说出来是很简单的，无非是在网站根目录下放置一个名叫favicon.ico的图标文件，然后就可以啦。

而这个favicon.ico文件呢，其实说白了就是16x16像素的小图片，现在浏览器原则可以支持GIF、JPG或者PNG格式的图片，但是为了更好的支持所有的浏览器，ICO格式(Microsoft icon format)依然是最好的选择。

可以通过如下在线工具来制作/转换ICO格式：

  • Favicon.ico & App Icon GeneratorFrom Dan’s Tools
  • Online favicon creator generator ico convertor

至于操作步骤按提示操作就可以啦，就不再赘言啦。

尽管对于有些浏览器，将favicon.ico放入站点根目录即可，但是添加如下代码将会更好的兼容各种浏览器：

<link rel="shortcut icon" href="/favicon.ico" type="image/x-icon">
<link rel="icon" href="/favicon.ico" type="image/x-icon">

比如steemit的页面头部就包含了如下代码(其中v=2我估计可能就是区别于之前的favicon)：

<link rel="icon" type="image/x-icon" href="/favicon.ico?v=2" data-reactid="22"/>

更多关于Favicon的使用详情，请参考文末链接的维基百科内容。

一个业务办多半天 / My bank account has been permanently frozen

之前的帖子里曾经说过我在一个银行的网银突然就被永久冻结了。


(图源 ：pexels.com)

我很确定我没用这个银行的银行卡做过什么坏事（呸呸，其实我没用任何一个银行卡做过坏事），不过这个卡就是医保扣款的，按说更应该没毛病才是。

因为要经常查一下流水和余额，所以我特意跑了一趟银行去办理解冻。银行的柜台工作人员操作了半天之后告诉我没问题啦，让我回家再试，然后我就相信了他们，结果回来一试还是永久冻结。

这一晃过去了三个半月了，昨晚一试，竟然还是被冻结，今天只好又跑一趟银行了。

到了银行，和大堂经理说了一下情况，然后被请进贵宾室，倒不是因为我是他们行的VIP，而是我这个业务很复杂。

柜台小妹一脸茫然地问我怎么个情况，我描述一遍后还是没听明白，总算听明白之后又问我之前帮我解冻的人怎么说的，我回答他们根本就没给我解开啊。

然后小妹就是各种查询啊，各种喊人啊，他们各种研究啊，我等得无聊去大厅转了几圈回来她们还没研究明白。

我建议她可以考虑把我的网银注销后再重新开通，小妹又开始研究如何注销网银。

就这样，从上午不到9点，一直办到中午11:30多一点，总算告知我把我的问题解决啦。

不过这次我是不会轻信他们啦，之前就是太相信他们害我又跑一次，我这次带着法宝来的。拿出我的笔记本，打开网银，登录了一下，哈哈，可以用了，这次总算没骗我。


(图源 ：pexels.com)

按说重置一下网银应该不是啥大事，结果这么一个业务竟然办理了半天，哎。不过尽管工作人员业务不熟，但是服务态度还是相当不错的，值得表扬一下呢！

阻止朋友陷入Plus token传销陷阱

几天前一个好友突然微信群要咨询我炒币的问题，问我是否了解Plus token for Asia.


(图源 ：pexels.com)

其实尽管周围好多朋友知道我混币圈，但是我真的算不上啥币圈人士，这么知名的项目我并没有听说过。

不过朋友继续和我说这个项目，说是能炒比特币，说是比特币马上就要翻翻了，说是智能钱包等等。我直接回复他：一听就是骗人的。

然后朋友继续发他所了解的所谓“资料”：

  • 钱包是由谷歌和韩国三星团队联合研发
  • 智能狗搬砖，月收益10-35%
  • 十层下线奖励
  • 其它各种优点

这次我可以明确地告诉他了：“这明显是传销！”

然后告诉他：“你要是钱多了没地方扔，可以给我一些，不要玩传销。”朋友表示他只是听他朋友介绍心动了，所以咨询我一下，既然我这么说，他就不买了。

然后的事情大家应该都知道了，这两天媒体铺天盖地的报道：币圈第一大传销项目Plus token崩盘跑路，卷款无数。

朋友微信上私聊我：“多亏咨询你了，不然惨死了，你真有先见之明啊。”

其实哪里是有啥先见之明啊，我炒币几乎都是亏的，哪里有那么好赚又稳赚的项目呢？况且判断传销太容易了，一个就是稳定的高额收益，再有就是下线机制。如果符合这两点，果断远离就对了。


(图源 ：pexels.com)

还有就是大体而言，投资的风险和收益往往都是成正比的，想一夜暴富，就要做好一夜赤贫的准备。当然了，如果运气特好另当别论了。

升级本地节点到EOS 1.8.0

昨天的帖子提到编译了 EOS 1.8.0，但是单单编译可还不够，还要将之前运行的v1.7.4版本的节点升级到v1.8.0才行。


(图源 ：pixabay)

不过因为这次是共识升级(硬分叉)版本，所以以往的直接替换是行不通的啦。

删除bnet plugin设置

首先我们要删除之前bnet相关设置，因为在v1.8.0中bnet已经彻底不被支持啦。

如果没有删除相关设置，运行v1.8.0版本的话，就会出现类似如下unrecognised option错误：

error 2019-06-30T01:39:04.740 nodeos main.cpp:120 main ] Throw location unknown (consider using BOOST_THROW_EXCEPTION)
Dynamic exception type: boost::wrapexcept
std::exception::what: unrecognised option ‘bnet-follow-irreversible’

删除相关目录

因为是共识协议升级，所以新版本的chainbase和之前版本chainbase是不兼容的，在运行新版本之前需要删除data目录下的如下内容：

  • blocks/reversible directory
  • state-history directory
  • state directory

如果没有删除上述目录就开始运行v1.8.0，那么就会出现如下错误：

N5boost10wrapexceptISt13runtime_errorEE: chainbase database format not compatible with this version of chainbase.

Replay

当做完上述工作后，再次重启v1.8.0的nodeos，系统会自动进行replay。

不过看起来，replay是一个漫长的工作，我这跑了约12个小时，才将将完成1/4，目测全部完成，需要2天以上。

升级指南中的步骤

把升级指南中的步骤附一份，供大家参考。

  • Ensure that their existing node is running the most recent stable release (1.7) of nodeos and then shut down nodeos.

  • Make a backup and delete the blocks/reversible directory, state-history directory, and state directory within the data directory.

  • Replace their old version of nodeos with the new release.

  • Start the new 1.8 release of nodeos and let it complete replay from genesis and catch up with syncing with the network. The node should receive blocks and LIB should advance. Nodes running v1.8 and v1.7 will continue to coexist in the same network prior to the activation of the first protocol upgrade feature.
