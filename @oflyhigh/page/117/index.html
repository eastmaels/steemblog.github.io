微信公众号增加RC查询功能 / Added RC query function to weChat gadget

RC系统(Resource Credit System)是在STEEM在HF20中引入的功能，替代之前版本中的bandwidth系统，主要目的是防止STEEM网络被滥用，为STEEM的可持续发展铺平道路。


(图源 ：pexels.com)

RC系统上线了很长时间，属实在反滥用方面起到很大的作用，但是也给一些新手带来不便，常常因为RC不足而无法进行发帖或者点赞操作。

为了方便大家查询当前自己的RC，我在微信公众号中添加了对应功能，现在大家可以很方便的查询啦。

查询方法

为了避免N多繁复的指令，我把RC查询功能直接放到账户查询功能中了，指令如下（将STEEMID替换成你自己的）：

@steemid

示例如下：

其中，RC为当前的RC量，MAC_RC为你的RC总量，Percent为当前RC的占比。

RC的总量

如果很擅于观察，那么会发现MAC_RC的数值和有效SP相当贴近，事实上也大致如此。

所以RC总量计算的相关代码如下：

RC的总量由以下因素决定：

  • 当前用户持有的vesting_shares
  • 减去用户代理出去的vesting_shares
  • 加上用户收到的vesting_shares代理
  • 加上max_rc_creation_adjustment
  • 减去下次Powerdown要变现的vesting_shares

其中别的项目相对理解简单，max_rc_creation_adjustment可能会让人费解一些，其实这个就相当于注册RC账户时注册费(STEEM形式)按当时状况计算的vesting_shares。

RC的恢复

RC按时间线性恢复，每日恢复总量的20%。

你可以将RC想象成一个水桶，水桶能装的水就是RC的总量，另外有一个水管往水桶里注水每天注入1/5，当水桶满了，水管继续注水，水从桶里溢出。

计算当前RC时，一个重要的问题就是要考虑系统给的RC的更新时间，需要根据当前时间和RC更新时间对当前RC值进行修正。

RC不够怎么办？

网络上流行一句话：你这个问题，充钱就能解决

所以最简单的方式就是Power up SP了，当然减少对外代理避免Power Down，或者能够让朋友代理给你一些SP，这都是可以解决RC不足的好办法。

公众号二维码

微信公众号继续欢迎大家关注，有很多方便的小功能，还在不断完善中。

终于，还是进行了ZB的身份认证 / Finally, completed the Identity Verification in ZB

前两天发帖子说ZB开始强制要求高级实名认证，不认证就没法C2C了。我在微信上和朋友说起这事，她强烈鄙视我，说C2C多贵呀，简简单单几步操作就可以把认证搞定，为啥就抵制呢？

仔细想想，好像也没啥必须抵制的地方，无非是被强制要求做什么事情，心里不爽而已。于是昨天抽时间按ZB的提示，一步一步把高级认证做了。

大致流程是，先拍身份证人像面，再拍身份证国徽面，然后录一段念ZB提示的4个数字的视频并提交，然后认证就通过啦。

再回来看，就会发现提示认证已通过。

然后尝试做一下OTC操作，发现要设置昵称、绑定收款账户、绑定手机等操作，终于都绑定好了

试着搞了一笔OTC操作

从下单到完成一共不超过一分钟，钱就到账了，另外手续费是C2C交易的十分之一，看来这验证也不白做啊，难怪被朋友鄙视啊。

新工具，SP委派查询 / Tool to check SP Delegations

SP委派 (SP delegation)我们也常常叫做SP代理，是STEEM硬分叉18之后引进的功能。


(图源 ：Pixabay.com)

简单地讲SP委派 (SP delegation)就是

委托人(delegator)将股权(vesting_shares)委派给受托人(delegatee)，股权（vesting shares）仍由原始账户（委托人）所有，但是投票权、投票收益以及资源(rc)分配等权益被转移(委派)给受托者。

微信公众号版

我之前已经在微信公众号中实现SP委派查询，比如查询我都代理给了谁：

但是微信公众号有返回结果的长度限制据说是2048字节。所以当返回数据内容过多，则公众号会返回如下信息：

网页版

尽管微信公众号用起来还算可以，但是为了方便不使用微信的朋友，或者SP委派数据量过大的朋友，我又弄了一个网页版。

网址为：https://www.eztk.net/delegations.php

显示结果分为两组表格，一组为委派出去的SP，一组为即将回收的SP。

补充说明

如果你很久之前代理出去一堆SP，那么当你再次检查你的SP代理时，你会发现一个有意思的事情，就是你的SP代理数值长大啦。

以我代理给@exec账户的数据为例

2017年6月的时候我代理给这个账户2000 SP，现在过去了快两年的时间，代理出去的SP变成了2071，足足多了71个。

这个钱哪里来的呢？答案是由STEEM的通胀产生并发放给SP持有者的，是不是很惊喜啊？

无耻地蹭到了草莓🍓 / My friend brought me fresh strawberries

春天是吃草莓的季节，上午还去超市买了两斤所谓的丹东九九草莓，回来后洗了一些吃，觉得味道还不错呢。

中午没事刷手机朋友圈，看到一朋友在草莓采摘园秀她的战利品，看起来就觉得一定比我买的草莓好吃呢。

看着这诱人的草莓，我忍不住点了赞。

下午突然接到朋友的电话，问我是否在家，我回答在家，她说你别出门，我给你送草莓。我说不用啦，你辛苦采摘的，留着自己吃吧，况且我上午刚买了一些。

她回答说，给你们带份啦，已经开车到小区附近啦，马上就到。

于是，我就无耻地蹭到了一箱采摘的新鲜草莓，洗了一点吃，口感比我买的果然要好很多。

我决定啦，以后看谁在朋友圈发好吃的东西，我一定要第一时间点赞，万一再蹭到点啥呢。😏
