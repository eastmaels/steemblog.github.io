终于吃到了传说中的白鹤大冷面 / White crane cold noodles

白鹤大冷面好不好吃？看排队和就餐的人群就可见一斑了。

所以我和朋友一直有一个梦想，就是要去吃一顿白鹤大冷面，不过每次总是因为排队的人太多而作罢。

今天朋友又拉我去白鹤大冷面，说一定要圆了吃白鹤大冷面的梦，所幸今天没什么事情，那就排队排着吧。

排了半天队，又费了好大的力气找座位，我们总算吃到了白鹤大冷面。

不过讲真，真没觉得白鹤大冷面很好吃，可能是因为我吃惯了经典的酸甜口味冷面的缘故吧。

不过不管咋说，总算圆了朋友这个梦，以后估计不会再喊我吃白鹤大冷面了吧？(^_^)

在Ubuntu 18.04上安装指定的node.js版本

尝试在一台Ubuntu 18.04的VPS上部署一个别人弄的项目，需要安装node.js以及npm等。可是使用Ubuntu自带的包管理器安装的node.js以及npm版本太低，导致部署项目时出一堆乱七八糟的问题。


(图源 ：pixabay)

旧的安装方式

安装nodejs与npm使用的命令为：

sudo apt install nodejs
sudo apt install npm

安装之后查看版本号：

node -v
输出为：v8.10.0

npm -v
输出为：3.5.2

新的安装方式

为了解决版本的问题，我搜索了半天，最终在NodeSource Node.js Binary Distributions: Debian and Ubuntu based distributions找到了安装指南。

curl -sL https://deb.nodesource.com/setup_10.x | sudo -E bash -

上述代码下载并执行setup_10.x这个脚本，这个脚本做了许多工作，比如说：

  • Populating apt-get cache…
  • Confirming “bionic” is supported…
  • Adding the NodeSource signing key to your keyring…
  • Creating apt sources list file for the NodeSource Node.js 10.x repo…
  • Running apt-get update for you…

执行完上述几项工作后，我们需要手动执行以下代码来安装Node.js 10.x 以及npm

sudo apt-get install -y nodejs

安装完成后查看版本号：

node -v
输出为：v10.16.0

npm -v
输出为：6.9.0

其它

尽管更新了node.js版本，但是项目我还是没有部署明白，哈哈，惭愧啊。不过依旧要每天进步一点，也不枉接触一次。

相关链接

  • https://nodejs.org/en/
  • https://nodejs.org/en/about/releases/
  • https://github.com/nodesource/distributions/blob/master/README.md#debinstall

每天进步一点点: 手动安装指定版本的MongoDB PHP Driver

在尝试安装某项目时，遇到了如下错误提示：

mongodb/mongodb 1.3.2 requires ext-mongodb ^1.4.0 -> the requested PHP extension mongodb is missing from your system.

但是通过Ubuntu包管理器安装的只有1.3.4版本即：php-mongodb_1.3.4，这不符合版本要求，所以只有考虑手动安装。


(图源 ：pixabay)

驱动兼容

在驱动兼容页面，我们可以看到如下兼容关心，因为我用的是PHP7.2，所以原则上可以安装MongoDB 的1.3.x、1.4.x或者1.5.x的PHP驱动。

尽管在https://docs.mongodb.com/ecosystem/drivers/php/ 这个页面中说明可以用 PECL或者 Composer来安装MongoDB PHP Driver，但是我没搞明白怎么选择版本。

手动安装

不过我在PHP.NET上找到了一个Manually Installing the MongoDB PHP Driver

git clone https://github.com/mongodb/mongo-php-driver.git
cd mongo-php-driver
git submodule update --init
phpize
./configure
make all
sudo make install

因为我要选择版本，所以还要在进入目录后，添加这样一句（比如安装1.5.5版本）：

git checkout 1.5.5

补充

除了从github获取源文件外，还可以直接从PECL的下载页面获取，地址为：
https://pecl.php.net/package/mongodb

想要1.5.5版本的话，直接执行下列命令即可：

wget https://pecl.php.net/get/mongodb-1.5.5.tgz
tar xzvf mongodb-1.5.5.tgz

其它手动安装的步骤基本不变。

结果

按上述手动安装步骤安装后，之前出现的错误提示就消失啦。可是整个项目我还是没有安全配置明白，完全在自己的技能树之外。不过前进一点是一点吧，想必终将会整明白的。

相关链接

  • https://docs.mongodb.com/ecosystem/drivers/php/
  • Installing the MongoDB PHP Driver with PECL
  • Manually Installing the MongoDB PHP Driver
  • https://pecl.php.net/package/mongodb

宏颜获水 & 最讲究的程序

这两天炒的比较热的话题之一是李彦宏演讲的时候被浇了一瓶矿泉水，尽管百度的广告竞价推广、百家号以及百度全家桶都挺恶心，但是往人头上浇水这种事情还是不对的。

说到全家桶，大家想必对那些不请自来进入自己电脑的程序深恶痛绝，除了不请自来之外，这些程序还长期霸占电脑，撵都撵不走，想卸载？如果你不是这方面的专家，那么基本没门。

我想当然的以为很多程序都是这样，稍微讲究一点的多说卸载时很方便罢了。可是今天无意中看到电脑提示Java有更新，并且点了更新按钮之后，我被惊呆了，原来世界上真的有讲究的程序。

看到没有，当我尝试升级的时候，Java升级程序竟然提示我超过6个月没有使用Java了，建议我卸载，如果以后有需要的时候再重新安装。并且卸载步骤也很简单，只要三步。

对比国内一些垃圾软件不然卸载或者制造很多障碍来阻止卸载，Java当之无愧是最讲究的软件啦。

再想想某全家桶，以及宏颜获水，嗯，不管咋说，往人头上浇水这种事情还是不对的，哈哈。😳

又突然想到，如果有人给马化腾浇水应该用啥成语呢？或许应该往脚上浇，这样就可以用马湿前蹄了😀，好吧，但愿我别成为一个伟大的预言家。
