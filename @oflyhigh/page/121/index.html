考古：李商隐很有可能是混币圈的

大家最近都知道币市行情不好，哎，我的狗币啥的都快跌没了，不但没做成狗庄（狗币的庄家）反倒被狗庄狠狠地收割了。看行情闹心，看看诗吧，于是就看到了李商隐，咦，我看了他的几首诗，突然感觉他很有可能和我们一样是混币圈的啊。


(图源 ：pixabay)

不过貌似属于入圈比较晚，然后被收割的那伙。

首先来看这首《夜雨寄北》

君问归期未有期，巴山夜雨涨秋池。
何当共剪西窗烛，却话巴山夜雨时。

这首诗应该是李商隐刚加入币圈时候写的诗，刚加入币圈，那时候各种币涨势良好（涨秋池），（女）朋友问他你啥时候回来上班啊？李商隐回答很含蓄，没有说粗暴的回复：打工是不可能打工的，这辈子都是不可能打工的。而是很委婉的说：

你问我啥时候回来我也说不定啊，现在各种数字货币涨势良好，等再涨涨，我赚了很多钱，就洗手不干了，回去陪你。

然后再看这个《昨夜星辰昨夜风》

昨夜星辰昨夜风，画楼西畔桂堂东。
身无彩凤双飞翼，心有灵犀一点通。
隔座送钩春酒暖，分曹射覆蜡灯红。
嗟余听鼓应官去，走马兰台类转蓬。

写这首诗的时候，我估计是在币圈小有斩获。我们好多币圈的男士都喊过一币一嫩模的口号，尽管最后喊这口号的都成了嫩模，但是当时赚了钱就要Happy的心思可见一斑。所以李商隐赚了钱，出去喝点小酒，高兴一下，也是可以理解的啦。

《登乐游原》

向晚意不适，驱车登古原。
夕阳无限好，只是近黄昏。

写这个的时候诗人是很纠结的，他知道币圈危矣，但是还是觉得可以搏一搏，虽然近黄昏但是夕阳无限好啊，搏一搏,单车变摩托。 赌一赌,摩托变吉普，哦对了，这次登乐游原（驱车登古原）是骑单车来的，如果再次换个吉普再来，就爽歪歪了。

再来看《锦瑟》

锦瑟无端五十弦，一弦一柱思华年。
庄生晓梦迷蝴蝶，望帝春心托杜鹃。
沧海月明珠有泪，蓝田日暖玉生烟。
此情可待成追忆，只是当时已惘然。

这个就更明显了，何为弦？其实就是币价的走势曲线了，什么5分钟线，30分钟线，小时线，日线，周线，这些就好像正弦曲线的波浪线一样，起起伏伏的啦。至于柱就好理解了就是K线图啦，一个棒一个棒。

李商隐写这首诗的时候，估计币市行情已然不是很好了，所以说一弦一柱思华年，就是说看到当前曲线K线跌得稀里哗啦的，就不禁回想当年形势一片大好的时候。至于后边的内容，还幻想回复往日荣光，但是他也知道这不现实，只能叹息此情可待成追忆了。

（PS: 这个时候诗人的奥迪已经变成奥拓了😭）

再来看诗人的《无题·相见时难别亦难》

相见时难别亦难，东风无力百花残。
春蚕到死丝方尽，蜡炬成灰泪始干。
晓镜但愁云鬓改，夜吟应觉月光寒。
蓬山此去无多路，青鸟殷勤为探看。

相见时难别亦难，东风无力百花残， 这句是说，没啥利好消息（东风），所有的数字货币都不景气啊。

春蚕到死丝方尽，蜡炬成灰泪始干，这个更好理解了，诗里边的春蚕以及蜡炬不过是韭菜的另外表达方式，而核心思想无外乎，被割得干干净净。

晓镜但愁云鬓改，夜吟应觉月光寒，头发都愁白了，被子都赔没了，能不觉得冷吗？哦，对了，房子也抵押出去了，露宿的好处就是抬头就能仰望星空。

蓬山此去无多路，青鸟殷勤为探看，啥也别说了，出家去喽，其实并非因为看开，而是出家了就可以解决吃和住的问题啦。


(图源 ：pixabay)

看吧，堂堂的大诗人因为炒币落得了什么下场。

所以年轻人们？你们还要继续炒币吗？
都别炒了，把钱借我，我去抄底，赢了五五分成，输了我就出家了。

(额，本文仅为调侃，对大诗人李商隐我是相当敬重的，不过借他的名气调侃一下，想必他不会介意的，阿弥陀佛~~)

双泉寺（望海寺）的石头

行走在双泉寺里，给我的一个印象就是字太多了。当然了字不可能凭空出现，都是刻在石头上，找这么多天然的石头，再刻上字，倒真是一项了不起的工程。

字我就不一一解读了，大家自己看图吧。


据说天气晴好的时候，站到最高处的小亭子上，可以看到远处的大海。可惜这天有些阴沉，我也没能看到大海（确切地说，我根本没有爬上去，懒啊）

偶尔放俩石头，写点文字能起到一些点缀作用，但是这石头和文字也太多了吧。

每天进步一点点：MFC Combo-box 突破文本长度限制

好久之前做了一个MFC+ACCESS的小程序，原本想着做好发布在STEEMIT上，结果发现一堆坑，一堆BUG。打算换成MFC+SQLite方式，却一直懒惰，没去更改。


(图源 ：pixabay)

说到BUG，其中一个是我所无法忍受了，我在程序中用到了Combo-box控件（CComboBox），然后模式设置为DROPDOWN，我可以在里边输入STEEM用户名也可以从以有的用户名当中选择。然后这两天当然想输入某个较长的用户时，用户名无法输入完整！

从迹象上看，控件多长就只能输入多长的字符串，然后就输入不进去了，似乎被限制了长度，这就尴尬了，STEEM上有好多较长的用户名，这样我的软件岂不是无法处理这些用户了。

是不是我在程序中不小心限制了这个控件的输入文本长度呢？查了一下，还真没限制。那么我可不可以在属性中设置它的长度呢？查了一下，没觉得哪个项目对应长度限制设置。

CBS_AUTOHSCROLL

于是去MFC文档中找找看有没有对应设置，最终让我找到这样一篇文档： Styles Used by MFC#Combo-box styles，其中有这样一个条目：

CBS_AUTOHSCROLL
Automatically scrolls the text in the edit control to the right when the user types a character at the end of the line. If this style is not set, only text that fits within the rectangular boundary is allowed.

咦，这不是正是我程序想要的功能吗？可是该如何设置呢？于是想了个笨方法，直接编辑.rc文件。在VS2015中选择对应的rc文件，右键点击，选择View Code，其中对应COMBOBOX 代码如下：

COMBOBOX IDC_COMBO_UPDATE,17,196,50,30,CBS_DROPDOWN | CBS_SORT | CBS_DISABLENOSCROLL | WS_VSCROLL | WS_TABSTOP

在后边加上个CBS_AUTOHSCROLL 试试：

COMBOBOX IDC_COMBO_UPDATE,17,196,50,30,CBS_DROPDOWN | CBS_SORT | CBS_DISABLENOSCROLL | WS_VSCROLL | WS_TABSTOP | CBS_AUTOHSCROLL

保存后重新编译程序，耶，搞定了，可以输入任意长度字符串了(65,535 bytes)，再也不怕STEEM上的长用户名了。

属性中设置

可是VS2015会这么弱智呢？这么简单的问题竟然要大费周折的去手动更改rc文件？我不相信！

于是我在仔细找对应控件的属性，咦这个Auto是什么鬼？

点了几下再去看我的RC文件代码，我晕，这个不就是设置CBS_AUTOHSCROLL的开关吗？

如果我之前看属性的时候耐心一些，挨个点一下，或者我的屏幕足够大，能在底部显示出每个条目的解释（之前窗口太小隐藏了），我就不必费劲找什么文档了。


(图源 ：pixabay)

哎，屏幕太小影响工作效率呀，看来有必要换一个4K显示器了。不过看看STEEM和SBD的价格，还是将就用我的滥显示器吧😔

入此门者得解脱

解脱门上的对联，我瞪大眼睛对照了好久，才分析出来是如下内容：

暮鼓晨钟警醒世间名利客
经声佛号唤回苦海迷路人

不过话说，世间名利客还能警醒吗？别说暮鼓晨钟了，估计高音喇叭也警不醒啦，或许大榔头大凿子敲敲打打能警醒吧。

经声佛号更难唤回苦海迷路人了，因为迷路人并不觉得 自己迷路啊，也不觉得自己是在苦海，相反的觉得甜着呢，觉得目标超级明确超有奔头。

哎，既然无法解脱，还是换个门进吧。话说这啥门啊，门上的字呢？

找到一处有大字的门，欲界天，按说欲界天已经属于六道中的天道了超脱了人道，但是欲界天人还是有各种欲念的，天人尚且如此，何况人乎？

看来想得解脱真的没那么容易啊，还是继续加仓吧~~

这个你敢进吗？（酆都城）

在牛头山双泉寺除了有好看的花花草草以外，还有这样一处地方，一看名字，就觉得阴森森的呀，话说这个你敢进去吗？进去之后还能出来不？

城门两边一幅对联：

度尽万生方证菩提
地狱不空誓不成佛

这其实是地藏王菩萨说的啦，地藏菩萨又被称为大愿地藏菩萨，据传在过去世中，曾经几次将在地狱受苦的母亲救出；并在久远劫以来就发愿要救度包括地狱众生在内的一切罪苦众生。

据说地藏王早就可以成佛的果位了，但是他发愿：地狱未空，誓不成佛，众生度尽，方证菩提，真是了不起！

扯远了，还是来看图吧

据说这里边有18层，不过我只在门口看了一眼，进18层地狱参观还是需要勇气的啊。怪吓人的，我还是闪了吧~~，阿弥陀佛。
