表扬一下京东

回想起来，似乎我没少在STEEMIT上吐槽京东，但是表扬，好像一次都没有。其实之所以大多时候我购物选择京东，还是因为它有它的优点，比如说商品正、送货快、服务好。

商品正，这个不用多说了，京东自营应该没有假冒商品（至少我没遇到过）；送货快，因为京东在全国好多地方有仓库，所以一般情况，东西可以第二天就收到，快的时候甚至当天可达；服务好，尽管有时候会遇到一些问题，但是大部分都圆满。

这次之所以提起这个话题，是因为大概3个月以前，我买了几个飞利浦品牌的E14接口的LED灯泡，后来由于种种原因只用了一枚，但是前两天这个灯泡突然就罢工了。

虽然有替换品，但是接近30元一枚的灯泡就这么报废了，还是有点郁闷，于是去京东的售后页面看了一下，点进对应的订单一看，呦，这灯保修3年呢，那就提个售后申请吧，之后我就忘记这事了。

结果不一会，电话响了起来，来电是950618，我还以为是哪个卖保险的呢，漫不经心地接了起来，原来是京东客服，问了一下我的灯泡情况，告诉我直接给我补发一个。

客服还不断地向我表达歉意，说因为这边仓库已经没有货了，要从外地调货，所以可能要2-3天才能收到，耽误我使用了。我连忙表示没有关系，我这还有备用的呢。

过了一会我去京东后台查看，看到服务单已经通过审核。

并且生成了新的订单（金额是0，不用付款哦）

一个灯泡倒不是很贵重，不过京东这快捷和周到的服务太让人觉得舒心了。连各种虚拟币大跌，都不那么让人觉得难受了😭😭😭😭😭

币市有风险，投资需谨慎

最近手欠，看账户里有点SBD，想了想，换成STEEM吧，于是在0.8SBD/STEEM附近，换了一些STEEM。前两天看跌到0.75、0.72，又随手换了一点，然后今天变成了0.65SBD/STEEM😭


(图源 ：pexels.com)

另外我人民币一毛以上（大概有0.128或者更多）的时候投了一点SEER，其实我并不知道这是啥，但是总觉得很便宜，买点或许赚钱呢，然后它一路跌到几分钱。

后来SEER上大B网，我觉得它应该雄起了吧，然后在6附近一路加仓，可是还没来得及出手，就跌了下来，既然如此就留着吧，然后一路跌到1分。前些天想着1分总不能跌破了吧，于是果断加仓，结果现在奔5厘去了。

昨天白天闲着没事，加了好多乱七八糟的币种，当然并未计划长期持有，想着只要稍微涨一点点，赚个差价就好。然后晚上，暴跌开启了……

看着几乎所有币种一起瀑布，尤其是自己加的币种更是走在瀑布的前端，那心情真的是无比酸爽，不过除了酸爽也没啥别的办法了。

既然已经瀑布，既然没后悔药可吃，既然现在已经无力进行什么操作（也许越操作越错），那么就找个好看的瀑布图来欣赏一下吧。


(图源 ：pexels.com)

顺便说一句，千万不要给别人推荐虚拟币，这几天看好几个群里不少人因为赔钱互相埋怨。

惭愧的是，去年有两个朋友总让我介绍有前途的币种，不好意思回绝，就介绍了几个我看好的币种，然后他们今年都不理我了😭….

虽然我介绍的时候一再强调币市有风险，投资需谨慎！

BUG? 在非不可逆块应用get_ops_in_block API 返回结果为空

在我的一个程序中，我尝试用get_ops_in_block获取指定区块的内容，但是我发现一个诡异的事情，如果是比较新的区块，这个API返回空的结果。


(图源 ：pexels.com)

经过我一系列测试，得出结论，只能对不可逆块应用get_ops_in_block，也就是说block编号小于last_irreversible_block_num的块。

一般情况不可逆块比最新块落后约20左右个区块（一分钟左右），如果对最新块或者这期间的20左右个块应用get_ops_in_block，就啥也取不到喽。

测试步骤 &结果

  • 获取最新块(head_block_number)

    {"jsonrpc": "2.0", "method": "condenser_api.get_dynamic_global_properties", "params": [], "id": 1}

  • 调用get_ops_in_block，其中xxxx用最新块(head_block_number)

    {“jsonrpc”: “2.0”, “method”: “condenser_api.get_ops_in_block”, “params”: [xxxx], “id”: 1}

  • 返回结果如下：

    {'id': 1, 'jsonrpc': '2.0', 'result': []}

补充说明

当然，这可能就是这样设计的，不过如果我没记错的话，好像HF19的时候不是这样的，当然也可能是我记错了。

但是如果就是这样设计的，那么应该在文档里加上说明，至少我在文档里是没找到相关信息。
https://developers.steem.io/apidefinitions/#condenser_api.get_ops_in_block

我在github上提交了一个issue，看看官方咋说😀
https://github.com/steemit/steem/issues/3155

今年雾霾特别轻

昨晚出去和朋友散步，突然想到一个问题，以往供暖季来临，雾霾也应该随之而来啊，而今年进入供暖季已经快半个月了，感觉这空气质量还不错嘛。

看了一下手机上的天气预报，其中各项空气质量(粉尘、PM2.5、AQI等)都是良好，虽然我不知道这些条目的具体含义以及标准，但是良好终归是好事情，另外，从我个人感受上来讲，空气清新没有异味，想必不会太差。

而1、2年前，每到供暖季来临，雾霾是常态，空气质量数据爆表那是经常的事情，空气中弥漫一股呛人的气味，出去转一小会回来的时候就会满面尘埃，空气净化器如果开成自动档，那就会几乎24小时不停地在最高负载下工作。

还记得3年前的我一个邻居朋友，在雾霾中散步的时候，突发感慨：“不行，我再也不能忍受这雾霾了，再也不饿能忍受这样的生活了！”于是两年前，她全家成功地移民到国外一个有碧水蓝天的城市。而我，只能感慨：“贫贱不能移！”

不知道大家是否还记得柴静那个《苍穹之下》视频，一度非常火爆，但是没几天就销声匿迹。销声匿迹的背后原因，我们不做猜测，不去探究，我只希望将来随《苍穹之下》消失的还有雾霾，还我们一个碧水蓝天。
