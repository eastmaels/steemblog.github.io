红高粱酒店

和朋友家年前小聚一下，很为去哪吃发愁，最后朋友提议说她家附近有一个红高粱酒店，味道啥的还可以，那就去那里好喽。

到了点菜的时候，我才发现这里的菜量都好大啊，价钱有有些贵，和我想想的精致好吃，略有差异。

不过既然来了，就点吧，于是随便点了几个菜。

当然了，还要给女士和小孩点上饮料，不过因为是开车过去的，男士没有啤酒。/(ㄒoㄒ)/~~

上菜喽，菜还是不错的，虽然菜量没有展示的那么大，但是也不少啦。

来个合影，哈哈，这样看起来，盘子小多啦。

总体来讲，环境和卫生都不错，菜品味道也可以，就是停车困难，我可是开出去好远，才在一条胡同里找到位置。

生日快乐

今天是1月14日，从昨天起登录建行等手机网银APP都会用巨大的闪屏祝福我生日快乐。


(图源 ：pixabay)

而今天早晨，更是收到了一大堆祝福短信，比如基金公司说要用他们的祝福带给我快乐成功的一年；证券公司祝我生日快乐，说遇到啥问题都可以找他们；保险公司祝我身体健康、合家欢乐；通信公司祝我每一天都快乐，等等等等

除了这些言语的祝福外，一些商场发短信通知我给我准备了生日礼包，有的送蛋糕，有的送小礼品，有的送积分，唯一遗憾的是要我自己去领取。

招行掌上生活的生日礼，有食品券和积分商城7折优惠券，这个7折优惠券可是太好了，原件10000多积分的商品，7000多积分就可以买回来了，而3000积分可是要刷6W才能集到的呢。

招行客户经理说给我准备了生日大礼包，让我哪天去网点的时候记得找他，不过不抱太大期望，一般都是锅具啥的😳

按说这么多祝福以及实物礼品啥的，我这个生日应该过得很高兴吧？可是问题是，我真的不是今天过生日啊。

其实说起我的生日到底是哪天，还挺复杂的，按说生日就应该是我出生的日子，可以上户口的时候就给我登记错了，月份上给我少写了一个月。

然后还是我上小学的时候，一次全国范围内的更换户口簿，又把我的生日在原来错误的生日基础上又加了10天。

我记得为这事我特意去找的户籍民警，可是人家明确的告诉我改不了。我据理力争，明明是你们给弄错的，为啥就改不了呢？人家告诉我就算弄错了，也改不了。


(图源 ：pixabay)

哎，所以我就在今天收到大量的祝福喽，不过收到祝福终究算是好事喽：）

每天进步一点点：Requests的超时(timeout)设置

在好久好久之前，曾经写过几篇关于Requests的文章，但是对于超时(timeout)却没有提及，最近看几个朋友讨论Requests的超时问题，所以特意学习了一下。


(图源 ：pixabay)

简单测试

为什么要讨论超时的问题呢？因为Requests链接或者访问数据时，并没有默认的超时设置，这就意味着，如果服务器端故障，导致无法连接等问题，那么Request就可能等待一分钟乃至更长的时间（甚至无限期地等待)。

我们可以通过如下代码进行简单测试：

requests.get('https://github.com:8888')

上边的代码等同于如下代码：

r = requests.get('https://github.com:8888', timeout=None)

实际测试，在等待几分钟之后方返回（时间应该取决于服务器的设置）：

Failed to establish a new connection: [Errno 110] Connection timed out’,))

而如果指定了timeout参数，那么当五秒钟过去，且并没有建立连接或者返回数据，那么就会返回Timeout错误：

requests.get('https://github.com:8888', timeout=5)

比如上述代码会返回类似如下内容：

‘Connection to github.com timed out. (connect timeout=5)’))

由此可见，设置timeout参数是多么重要。难怪Requests文档中提示：Nearly all production code should use this parameter in nearly all requests.

高级设置

朋友在微信群中转发过来的一篇文章中提及：

requests请求上设置的timeout判断的并不是整个请求的总时间，而是从与服务器连接成功后，客户端开始接受服务器的数据为计算起点的。

为此我特意找了一下Requests的文档，其中关于timeout参数的说明中说到：

The timeout value will be applied to both the connect and the read timeouts

也就是说，朋友转发来的内容要么是错误的，要么是过时的。（所以看资料还是应该去官网）

除了用一个timeout指定超时参数外，也可以分别指定连接以及读取数据的超时时间，指定方法如下：

r = requests.get('https://github.com', timeout=(3.05, 27))

关于连接超时时间，从经验上看，一般设置为比3略大的数字，详细原因如下：

It’s a good practice to set connect timeouts to slightly larger than a multiple of 3, which is the default TCP packet retransmission window.

没想到小小的timeout参数还有这么多说道啊，看来以前了解的果然肤浅。

秃顶恐惧症

要说中年人尤其是程序员最容易遭遇也最怕的事情当中，秃顶绝对是可以排得上名次的。


(图源 ：pixabay)

网上曾经流传一张程序员的技术等级图，

没错，通过头发的稀少程度，就可以判断一个程序员的技术等级啦。

几年前，为了让自己显得不像是一点水平都没有的样子，我决定将头发剃光，变得比代码救世主更加救世主的样子。然而这样做无疑是聪明绝顶和自作聪明的区别，我的代码水平一如既往的烂。

后来我楼下的阿姨不止一次对我说，好好一个年轻人，弄什么光头，看起来和劳改犯似的，于是我又开始留头发。

不过虽然头发留起来了，看起来不是那么秃，但是自己的事情只有自己知晓，无非是长长的头发挡住了秃掉的区域。每天洗脸的时候，照照镜子都在想，虽然现在正面看起来还好，但是什么时候会秃光呢？

昨晚做了一个梦，梦中和朋友们一起聚会痛饮，然后在一群秃顶和光头中我的头发看起来很是“丰满”，让我甚为满意。结果过一会一个朋友分享了一个从后边照的照片，如果前面看是森林的话，后边已然秃成了沙漠。


(图源 ：pixabay)

被吓醒后，才发现自己对秃顶竟然如此恐惧，到了做噩梦的地步了。摸摸后脑勺，头发还在，心里安定了一点点，哎，还是好好珍惜头发还在的时光吧。
