哈哈，原来`sudo rm -rf /`没那么危险嘛 （注意，勿尝试）

今天想关闭一台VPS，随手清除一下VPS上的数据，其实按说这个完全没有必要，因为主机商文档说我们关掉VPS的时候数据就会被清除——况且我的VPS上也没啥敏感数据。


(图源 ：pexels.com)

不过闲着也是闲着，我就想起了江湖上赫赫有名令人闻风丧胆的命令：

sudo rm -rf /

话说，尽管没少听别人说过这个命令，也没少和别人说过这个命令，但是我还真没用过这个命令呢，今天就来试试它的威力如何吧。

我胆颤心惊地输入sudo rm -rf /回车并按提示输入密码以后，竟然提示我如下信息：

rm: it is dangerous to operate recursively on ‘/‘
rm: use –no-preserve-root to override this failsafe

翻译过来大致意思就是

在/(根目录)上进行递归删除是很危险的事情。
使用–no-preserve-root来强制执行

然后，新组合出来的命令就是这个样子喽

sudo rm --no-preserve-root -rf / (注意勿随意尝试)

然后，我的系统就崩溃鸟，我被从终端踢出来鸟，就再也登不上去鸟。

所以说，其实直接用sudo rm -rf /是没那么危险的，当然，前提是你的系统不是那么旧，至于多旧才算旧系统才能直接用这个命令，这个我懒得研究也不想研究，话说，研究这个得多无聊啊。

再次提示：无论是sudo rm -rf /还是sudo rm --no-preserve-root -rf /或者是root用户下直接运行rm -rf /，如果你不清楚你在做什么，请勿尝试。

不要依赖新系统给你所谓得保护，而是一定要清楚自己在做啥。如果你胡乱尝试造成系统崩溃或者数据损失，俺概不负责哦。
