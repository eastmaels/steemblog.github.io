今天注意到EOSIO在[github](https://github.com/EOSIO/eos)上释出了新版本EOSIO v1.8.3，话说这个版本推出间隔有些短，主要是因为v1.8.2中存在BUG，导致异常的高带宽占用。

不幸的是，前段时间手欠把本地EOS节点升级到了v1.8.2 确实也遇到了高带宽占用这个问题，那么只好升级喽。

# 升级EOS

![](https://cdn.steemitimages.com/DQmQDsC8e6KSnNR7vo6fk9UDprqQph4SS6gFeY2fwfjFZk2/image.png)

所幸编译这些东西已经轻车熟路啦，卡卡卡一通命令执行下来即可，至于编译程序用多长时间，那又有什么关系，耐心等待就好喽。

编译完成后检查一下版本号：
>`nodeos --version`

返回如下信息，版本号v1.8.3，看来没搞错。
>![](https://cdn.steemitimages.com/DQmbXoYutbQUHz2jogPUS3U3EMy6nWxouNC9ao7G4Aixa8s/image.png)

# 弃用`network-version-match`

v1.8.3 除了解决一堆bandwidth和p2p(Net plugin)的BUG外，最重要一个修改就是弃用了`network-version-match`选项。

这个限制方向很有趣，原本我以为是p2p限制其它不同版本的节点链接，后来才搞清楚是限制节点连接不同版本号的P2P节点。

不过现在不用晕啦，这个选项已经被弃用了。

# V2.0.0 版本？

在[EOSIO v1.8.3 Release Notes](https://github.com/EOSIO/eos/releases/tag/v1.8.3)，讲述弃用`network-version-match`选项时提到在v2.0.0版本：
>Setting this option to false is especially important given that the v2.0.0 release of EOSIO will add a new supported version of the network protocol that enables additional capabilities...

看来下个大版本有可能就不是v1.9.x了，而是直接v2.0.0，另外，这个大版本已经会引入更多的新功能和新特色，好期待。

# 共识升级

另外一点就是，距离1.8共识升级，还有不到10天啦。详情参考：[1.8: Consensus Protocol Upgrade Process](https://developers.eos.io/eosio-nodeos/docs/consensus-protocol-upgrade-process)吧。
