今天在一台新的机器上运行EOS的命令行钱包，遇到了一些问题，记录下来，方便自己以后查找，也希望能给遇到同样问题的朋友一点帮助。

![](https://steemitimages.com/DQmRkLq6rRew3mHfx4vYGWyqpC8wSebLPeC2iZCXAdpuGkR/image.png)
(图源 ：[pixabay](https://pixabay.com/))

将编译出来的命令行工具(v1.8.1)复制到新机器上后，尝试解锁钱包，出现如下提示:

>`cleos wallet unlock`

>"/home/test1234/bin/keosd" launched
>Unable to connect to keosd, if keosd is running please kill the process and try again.

![](https://cdn.steemitimages.com/DQmQt4GgE5q4vV2xmwTKTrjyTGZX7TupUEjpnnwFCGruCHu/image.png)

咦，既然已经***launched***，为何还连不上呢？尝试直接启动keosd试试看：
>`keosd`

提示如下：
>keosd: error while loading shared libraries: libcurl-gnutls.so.4: cannot open shared object file: No such file or directory

![](https://cdn.steemitimages.com/DQmcCxJ72r5VvJTp1LH1NvzcVUsSCba59b7kEHJbfgTkzwc/image.png)

原来keosd根本没有启动成功嘛，所谓的***launched***其实是骗人的😡

不过既然知道了原因，就好办了，缺啥装啥呗。
 `sudo apt-get install libcurl3-gnutls`

装上之后运行keosd又出现如下提示：
>keosd: error while loading shared libraries: libusb-1.0.so.0: cannot open shared object file: No such file or directory

![](https://cdn.steemitimages.com/DQmVsgGpAfGvoDPsyy7WPf9c1edBdVaaZS9CvQTo2rAjYk1/image.png)

继续暴力解决：
>`sudo apt-get install libusb-1.0-0-dev`

之后再运行keosd：
>`keosd`

一切正常：
>![](https://cdn.steemitimages.com/DQmREwzsDLNBusUCvncMmzUWC5XTxCLw8fFP7pNxwaW9bEB/image.png)

再解锁钱包试试：
>`cleos wallet unlock`

![](https://cdn.steemitimages.com/DQmZvrLCvh57pb1xwnjUxxBkGut6U1HdV9TCNWBGodFvCDe/image.png)

终于出现了密码提示窗口，看来这次***`launched`***是真的啦。
