最近基于STEEM的SCT平台火爆CN区，大家除了发帖外也积极给sct帖子点赞。但是有一个问题一直搞不清楚的是，SCT平台上点赞的VP和STEEM平台上的VP是否一样呢？

根据ScotBot的设置参数
vote_power_consumption
vote_regeneration_seconds

ScotBot的设置是需要不同的VP的

### 那要怎样看自己的VP呢？

昨天在steemcoinpan.com上看帖子，看到这篇帖子[내 SCT 보팅파워는 어떻게 될까?](https://www.steemcoinpan.com/sct/@philhyuntd/7tw1xn-sct)

虽然我看不懂韩语，但是图片我看的懂啊！这不是看VP的api吗？

API链接：https://scot-api.steem-engine.com/@用户名

比如我要查我在各个SCOT的平台的一些信息，就可以用这个api查看：https://scot-api.steem-engine.com/@ericet

这是从api得到的一些信息：

~~~
{
	"SCT": {
		"downvoting_power": 10000, //踩的能量
		"last_downvote_time": "2019-01-01T00:00:00", //最后踩的时间
		"last_vote_time": "2019-05-23T14:43:54", //最后点赞的时间
		"name": "ericet", //账号名字
		"pending_token": 28676, //待结算的收益。SCT是小数点后3位，所以是28.676 SCT
		"staked_tokens": 88874, //Stake的代币数量。88.874 SCT
		"symbol": "SCT", //代币的符号
		"voting_power": 6409//点赞VP
	}
}
~~~

从上面的信息，我知道我的点赞VP还有64.09%，待结算的SCT有28.676个，总共Stake了88.874 SCT。

虽然SCT平台上的VP和STEEM平台上的VP不同，但是他们仍然使用相同的点赞比重（Voting Weight)

所以你在SCT上点赞帖子，同样耗费STEEM上的VP。

steem-engine团队有计划对这个问题进行改进，未来在SCT平台上点赞就不会耗费STEEM上的VP了

最后感谢SCT的平台，没有这个平台前，我很少会去看韩语的帖子。但是有这个平台后，发现kr社区讨论的很多事情CN区是不会讨论的。比如SMT vs SCOT。感觉发现了新世界！
