steemit下多用户维护同一个公共账户


　在开始steem-python操作之前，首先确认你的系统已正确安装了steem-python，如果没有或不清楚，可以参考我先前的文章《steem-python的安装和配置》。




　　多用户管理账户



　　在steem-python上实现多用户用同一公共用户发帖(操作)，主要有两步骤：




　　1>将主账户ID的Active私钥加入到steem-python


　　　　steempy addkey 回车


　　　　按提示输入主账户的private key，如post private key 回车


　　　　输入你的walletpassphrase，第一次时系统会让你设置，记住这个密码，后续所有重要操作都需要这个密码确认身份。　　　　



　　　　如果你不想一个个私钥添加，可以用账户快速导入私钥，如下：


　　　　steempy importaccount xxx




　　　　其中 xxx为你要导入的主账户名称，回车后按照提示输入你的私钥即可


　　　　


　　　　查看导入的key信息是否成功


　　　　steempy listkeys 或者 steempy listaccounts


　　　　


　　　　2>将目标账户的公钥加入到steem-python


　　　　steempy allow –account xxx YYY 回车后输入你的钱包密码，确认后即可。


　　　　正常后会返回类似如下图信息：


　　　　


　　　　其中 xxx为公共账户，YYY为目标账户对应的公钥




　　　　所有账户的公钥，均可以在steemd.com/@xxx 上查到，xxx为你要查看的账户id


　　　　注意，上述操作需在同一行一次输入完成，参数要全，我在先前输入时忘记输入最后一个参数，导致一直无法得到正确结果，但是诡异的是，steem-python仍提示我输入key 和 钱包密码，最后给出出错提示。


　　　　完成以上操作后，你就可以使用公共账户ID身份操作了，见下图：   　　　




　　为什么要做多用户管理操作？

 


　　1、对于使用者用户，比如茶馆的店小二们只需要使用茶馆的账号laodr和自己的post private key登陆就可以发帖了，无需再记茶馆的post private key，尤其是当茶馆的private key更改时，也无需通知店小二做任何修改，可谓是大家都很方便；


　　2、使用多用户管理，可以识别每个用户的操作信息，反之，如果大家都用同一个账户和postkey，有点像大锅饭，即便有问题都不知道是谁的问题。

steem-python的安装和配置


　　关于steem-python的资料非常少，我在网上搜索了下，百度几乎没有，而google也只找到了如下这份文档：



　　就拿着这份我认为很粗略的文档，我开始了研究steem-python的艰辛过程，在花了一整个下午和上午后，基本算弄明白了其安装和基础命令操作。如果你也对steem-python感兴趣的话，可以参考下我写的这篇文章内容。


　　在开始正式内容之前，我们先来了解下steem-python可以做什么？


　　简单地讲，利用steem-python，可以操作任何你在steemit网站上的一些比如upvote/downvote、follow/unfollow、transfer、powerup/powerdown、buy/sell等，除此之外，还可以设置多用户管理同一账户发帖等，比如我们即将新开始的茶馆，因为将要聘请多个店小二进行共同管理，每个店小二需要独立发帖，这个时候利用steem-python就可以完成上述这些管理操作了。


　　当然，steem-python的功能不仅如此，我看了下文档，利用steem-python可以进行编程，甚至还提供api等，虽说文档做的很粗糙，但功能还是较为全面的，对此有兴趣和时间的朋友，可以继续研究哦。


　　要想安装和配置steem-python，你需要首先安装python环境，看了下官方文档，提示如下：


　　steem-python requires Python 3.5 or higher. We don’t recommend usage of Python that ships with OS. If you’re just looking for a quick and easy cross-platform solution, feel free to install Python 3.x via easy to use Anaconda installer.


　　大意是安装的Python需要至少3.5版本或以上，官方不推荐使用系统自带的python，并且建议使用Anaconda下的python安装。


　　这里注意，强烈建议使用官方推荐的Anaconda下的python安装环境，包括如果你有安装旧版本，也建议卸载后安装新的。我刚开始使用时没注意这个，导致出现一些莫名其妙的问题。


　　以下我会详细介绍具体的操作步骤，重要步骤我会用粗体注明，而具体操作命令部分，我会用斜体来注明，以方便大家阅读和操作。




　　1、安装Python环境


　　1>下载


　　去Anaconda下载Python安装环境，地址如下：


　　https://www.anaconda.com/download/


　　比如我使用的是centoslinux6.4环境，以下同操作：


　    wget https://repo.continuum.io/archive/Anaconda3-5.1.0-Linux-x86_64.sh



　　注意，选择适合你自己的版本




　　 2>安装


　　 chmod+xAnaconda3-4.4.0-Linux-x86_64.sh


　　 ./Anaconda3-4.4.0-Linux-x86_64.sh



　　






　　 3>验证安装是否成功


　　 python-V


　　 返回：


　　 Python3.6.4::Anaconda,Inc.



　　2、安装steem-python


　　 1>>>安装steem-python


　　 pip install steem



　　 2>验证是否安装成功


　　 steempy–version


　　 返回：


　　 steempy1.0.0　　



　　好了，大功告成，我们终于安装好了steem-python，接下来就可以正式使用它了。


　　下一篇我会讲下steem-python 的命令行具体如何使用和操作。

只有懂得尊重用户的企业，才能赢得用户的尊重


　　全球最大的社交平台网站Facebook近日深陷数据泄漏丑闻中，据说事情的原因是因为facebook的不当数据共享给第三方企业，引起数千万用户数据被泄漏。




　　就在昨天周二的时候，Facebook的老总扎克伯格出席了美国参议院司法委员会和商务委员会联合举行的听证会，就公司对用户数据的处理接受了4个多小时的提问，当面承认了其公司的错误，作为一个国际知名互联网巨头企业老总，言谈诚恳，没有推诿，这点令我觉得很佩服。



pic source from PBS News


　　近来疯传的一些国内知名科技企业，利用自己的大数据优势，肆意使用用户的隐私数据等，甚至不乏一些企业利用老用户的数据来专门设定程式抬高价格，比如某国内知名旅游订票平台，当用户订票下了订单再取消后再次查询时，系统会自动将票价故意抬高到一个很高的价格，甚至制造出余票紧张的供不应求假象，让用户误以为一票难求而在不知情的情况下着急支付购买；




　　又有国内某知名打车软件企业，当老用户每天在固定的线路打车时，系统会自动识别老用户身份，故意加多显示费用，而同样的起点、终点和线路的其他普通用户价格却反而低过这些老用户的价格，利用大数据故意杀熟老用户，不能不说国内这些企业的一片用苦良心。




　　对于用户的隐私数据，国内某知名巨头企业在前不久的重要会议上甚至公开称国人对隐私数据不敏感，可以多多利用这些数据做更多的事情等，可谓无耻和荒谬到极致。




　　Facebook的数据泄漏一事，姑且先不说这次泄密其实也并非主观故意，但美国法院仍然作出了判罚上万亿的天价罚款，其罚款之重，背后的教训不能不让人深刻印象。




　　反观国内企业的做法，出现问题后首先往往不是诚恳认错，而是推卸责任，试图将自己撇的一干二净，当事情实在无法逃避时，又开始装聋作哑，甚至不乏一些低级旁门左道做法如雇佣大量水军、试图销毁视频硬盘数据等，唯独就是很少去正面认识和承认错误。古人云，知错能改善莫大焉，做人知错下次改正，企业也一样，只有真正认识到自己的问题错误，下定决心以后避免再犯，相信老百姓多数都是可以原谅的。




　　但是很遗憾，国内这些大企业往往不会这样去做，这也就出现了一个奇怪的现象，很多国内企业做的很大，无论是规模还是影响力，有些甚至成为行业的寡头，用现在流行的词语叫独角兽，按理说这样的企业能做到今天这个程度，一定是得到人们长期的认可和欣赏的，但在国内却偏偏不是，很多大企业反而更容易遭受老百姓的唾弃，这背后的原因显然大家都是明白的。




　　我很赞同美国法院的判罚，要么不罚，要么就重罚，而不是像国内一些做法，如股市非法交易操作，获利千万上亿，却最终只被罚款几十万而已，这样的处罚对于作恶者来说根本就是九牛一毛无足轻重。




　　我们不去讨论这罚金相关内容，对于美国法院的这种判法，我也是很认可的，无论企业是有意或者无意，只要作恶了，就应该明确高额处罚，杀一儆百，重罚之下也会让其他一些企业今后在类似事情上不敢再妄为，彻底断绝再做恶的念头。




　　马克思在《资本论》里说过这样一句话：为了100%的利润,它就敢践踏一切人间法律；有300%的利润,它就敢犯任何罪行,甚至绞首的危险。




　　作恶的成本很低，换来的收益却可能很大，而风险极低甚至没有，这便是国内互联网隐私环境的窘相。试问在这么高的收益却极低的处罚之下，换做是你，如果有机会的话，难道你会不心动？即便你一开始能忍住不作恶，但若你的同行甚至竞争对手都在利用它来谋取利益，你还能继续坚持多久呢？




　　只有懂得尊重用户的企业，才能赢得用户的尊重。


　　片尾插播两条最新新闻：


　　据悉，因为此次数据泄漏事件，Facebook已开始推数据滥用悬赏计划：最高奖励4万美元


　　百度回应Facebook数据泄密事件：我们把安全隐私视为最高原则 


　　一正一邪，大家心里自有评说，看透不说透，走过路过，笑而不语。
