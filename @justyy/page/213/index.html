Steem API - Get Curation Stats 查看你平均的点赞数据


Image Credit: atlassian.com

How much did you curate per day? You can see curation rewards for the last 7 days via (Rewards -> Curation Rewards)

：

I spent some time providing this handy API to get the curation states i.e. replace the ID part in the URL

https://uploadbeta.com/api/steemit/account/curation/?cached&id=justyy

It returns the JSON data

{"24hr": 2.8020695257949124, "avg": 2.6121824377557816, "7d": 18.285277064290472}

that corresponds to your curation rewards for the last 24 hours, average per day and last 7 days.

You can pass the value via a more secure POST method, like this.

curl -s -X POST 'https://uploadbeta.com/api/steemit/account/curation/' -d 'id=tumutanzi'

More SP, more curation rewards
{"avg": 96.34443079687016, "24hr": 91.77042842951606, "7d": 674.4110155780911}

You could use the following four SteemIt API servers (availability 24/7) globally (free of charge, subject to fair use policy):

  • East USA: https://helloacm.com/api/steemit/account/curation/
  • Japan: https://happyukgo.com/api/steemit/account/curation/
  • UK: https://uploadbeta.com/api/steemit/account/curation/
  • West USA: https://steakovercooked.com/api/steemit/account/curation/

这两周英国TESCO超市20%员工折扣，媳妇给孩子买了PS4游戏机。

我记得小学时候，老师说会玩游戏的孩子都很聪明。我们那个年代，小时候玩8位任天堂游戏，什么超级玛利通关暗道，什么魂斗罗30条命到现在都能耳熟能详。那个年代我们玩游戏得偷着玩，和父母商量着玩。什么考试多少分能玩多久又或作业写完了能玩一会。哪像现在的作父母的我们主动为孩子买游戏机。

上次去教堂，和教父聊天的时候就说到孩子有点注意力不集中，于是就分析到看电视看太久。孩子看电视是单方面的吸收，电视画面色彩鲜艳，很容易就被吸引过去了，久而久之，其它的事情就没那么能让孩子感兴趣而专心了。


看电视也是被动的吸收，据说也会影响孩子的创造力，所以还是尽量别让孩子看电视。这次媳妇从超市直接拎了一个回来，自带FIFA 18，游戏逼真，这足球是我最喜欢的足球游戏，但是很久没玩了导致一些操作都不太熟悉。虽然这游戏光盘上写着3岁以上就能玩，但是我觉得3岁要是能玩懂真的很牛。

玩游戏是交互式的，特别是在游戏的过程中，孩子也能学习、进步和成长。但凡事都得有个度，只要不沉溺于游戏，我就觉得没啥问题。至少比看电视强百倍。

不管如何，两熊孩子还是很感兴趣，手柄也摸了按了，还指着屏幕说哪哪哪个是我。不得不说，时代在进步，现在的游戏特效真是逼真，射门的时候手柄还能震动，游戏里的角色按真人定作，比如那个C罗简直很逼真的不是我们那个8位相素年代可以相像的。PS4游戏机还能上网接WIFI，吸入式光驱，连手柄都是无线的。不得不说现在的孩子真幸福。

试用 utopian.io

从 @jubi 菊老板的帖子中了解到这个 utopian.io，感觉特别有意思，于是昨天抽空试了一下。

utopian 是 乌托邦的意思。

utopian.io 是需要 steem 的 active key 的，posting key 不能登陆。而这个 active key 是可以用来转帐的，所以刚开始有点小纠结。

utopian.io 可以看做是基于 steem 区块链的一个定制化前端。utopian.io 主要是用于开发者提交代码、文档、BUG、分析、翻译等的一个专门的平台。

我试着在github 创建了一个代码库，然后按照需求在 utopian.io 上填写 Development，Github 仓库地址，还有就是代码库的说明，然后点击发布后：

  1. 文章已经第一时间发表在了 steem 区块链上，就和平时写文章一样，在 steemit.com, steemd.com 等都是可见的。文章地址是：Simple Video Download Helper
  2. 文章默认在 utopian.io 上需要被 moderator 审核后才能在 utopian.io 上列出来。
  3. 点赞收益数据在 utopian.io 和 steemit.com 上是同步的。
  4. 发表的文章自动加上了 utopian-io 第一标签
  5. 发表的文章自动是 100% POWER UP。
  6. 第一次在 utopian 发文后审核通过后，被 utopian-io 狠赞了43% 大概11美元，感谢鼓励！

由此可见， utopian.io 也是 STEEM 区块链的一个应用，只不过多加了一个审核还有代码归类的作用。

utopian.io 的用户界面，列出了所有 utopian-io 标签代码类相关的文章。

我在 utopian 上的主页地址： https://utopian.io/@justyy 是不是和 steemit.com, busy.org 类似，只不过这个 utopian 上专注于对开源项目有贡献的文章（所以请不要在上面发不相关的旅游贴之类的）
