一顿饭与 一个 C++ 软件工程师的职位

今天晚上，公司两大股东（其中一个是英国总公司的HR，另一个是中国公司的总经理）请我吃饭。我本来以为我犯了什么事呢，提心吊胆，结果原来是公司为了招人而犯愁呢。


鱼烤焦了

我们公司最近在招一个 C++ 软件工程师 （C++ Software Technical Expert），但是一直招不到合适的人选。但项目启动迫在眉睫，所以想听听的我意见。客观来说，我们公司的待遇还是不错的，但是C++的牛人一直招不到有一大原因就是：剑桥市有很多很牛的大公司，比如苹果、微软。能力强的很多不会到剑桥郡的一个村庄里上班。

先说好处：

  • 我们公司给办工作签证（五年英国绿卡），所以如果你想来英国工作或生活，这个机会难得，因为据我所知，英国并不是很多公司愿意给外国人发工签。
  • 我们公司属于发展迅速的高科技公司（无线通讯规划优化），你来了不像大公司里是螺丝钉，在我们这里可是会身挑大粱的，甚至可能会 Full Stack。
  • 灵活的工作时间，生活质量会更高一些。
  • 很有可能你的孩子将来上剑桥大学（至少物理距离更近了）
  • 你会遇到很多能力很强的人（不一定要在 Google, Facebook里，我们公司牛人也很多），能学到不少。

具体职责：

  • 用C++重构一些老的代码、模块、重写一些算法。
  • 改进开发流程。
  • 预言新算法，解决困难的问题。
  • 领导小开发团队。
  • 成为相关模块的负责人。
  • 撰写高质量的设计文档。

要求：

  • 很强的C/C++程序设计能力：需要对多线程、分布式、数据结构与算法都有很深刻的理解和相关经验。
  • 使用 GIT 的经验
  • 有领导团队的经验
  • 有过参于开源项目的经验 e.g. github
  • 熟悉 敏捷开发流程

Abstract: I have added a paramter named rep to filter out members in the wechat group that have reputation lower than the given value (by default rep=25). It allows us to focus on good quality contents. This parameter can be used in JSON API, RSS Feed for both posts and comments, Daily Ranking Table (and API) and the Web Reader.


Image Credit: Pixabay.com

声誉是积累起来，并且不能直接用钱立马买来的，积累声誉很慢，毁掉声誉却很容易。新注册的用户 声誉是 25级，如果干了些不明智的行为（比如抄袭）那么就会被踩，声誉就会很低甚至是负分。

所以，不管如何，一个人的声誉的确很大程度的反应他/她的文章的质量。声誉差的用户文章很有可能会被自动隐藏。我们想过滤掉低声誉的人的文章。说这么多，就是想给大家介绍一个微信群RSS（及API和阅读器）的参数： rep。默认是25，意思是低于25级声誉的数据将会被隐藏。

比如：https://helloacm.com/tools/steemit/wechat/ 默认不显示 低于25级的成员（并不是加了微信群就可以轻松上榜的）
当然，你可以加上 参数 rep=20 来显示大于等于20级的所有成员资料:

https://helloacm.com/tools/steemit/wechat/?rep=20

这有啥用呢？比如我就只想看等级高的文章，那么我可以：

https://helloacm.com/tools/steemit/wechat/rss/?rep=70

只看70级以上的文章。

这个参数适用于 JSON API, RSS (文章和评论), 微信排行榜，还有WEB 阅读器的URL地址。

说说英国政府提供每周免费30个小时幼托福利

这学期开始，我小儿子每周上幼儿园30个小时。英国政府给予父母双方都有工作（每周工作至少15小时）的家庭提供免费每周30个小时的幼托福利， 这样一来我们的经济负担就小很多了。


孩子幼儿园外写着免费30小时的福利这个月开始了

英国政府对于三类家庭有着不同的照顾：

  • 低保：家庭总收入较低，孩子从2岁开始就可以申请15/30个小时每周免费幼托。
  • 普通家庭： 一般家庭孩子3岁开始可以申请15/30小时每周免费幼托。所有家庭都可以，申请15小时不受签证类型影响。申请30小时需要让身份不受限制一方父母申请（比如拿了英国永居的我）
  • 高收入家庭：父母有一方年收入超过10万英镑，这种情况下是不能申请免费15/30小时幼托福利的。

幼托费用大概从每小时3英镑到10英镑不等。我儿子每周上幼儿园大概每周40小时，一个月就是160小时，这样费用大概是1000英镑左右，政府现在补助30小时，我们只需要支持200多英镑的学费。再加上公司有福利可以在税前把这部分费用给扣除了，这样的话税等于少交了，负担就没有那么重了。孩子4岁之后就上小学，学费免费，而且前3年英国政府免费给孩子吃午餐，怪不得在英国 养孩子压力并没有那么大。

我想英国政府这一政策是为了鼓励妈妈们出去工作，解决了家庭的后顾之忧。因为之前，如果妈妈们出去工作，那么孩子就得上幼儿园，而一个月有可能妈妈们的收入还支付不起昂贵的托儿费用，这么一来，很多妈妈想工作却不得不在家看孩子，这其实是不利于妈妈和孩子的：

  1. 研究证明，孩子到幼儿园能更好的发展和成长。
  2. 妈妈们也不容易和社会脱节，不至于一生孩子就放弃自己的事业。
  3. 英国是不能把低于12岁的孩子独自一人放在家里，如果被发现，孩子是会被政府领走并且由社区人员供养。情况严重父母是要不回孩子的。在英国，孩子是政府的（所以政府给发各种福利和奶粉钱），父母只是暂时代为养之。
  4. 英国文化是自己的孩子自己养，孩子的爷爷奶奶是很少会帮着养。很可能爷爷奶奶这时候生活是很舒服的，拿着退休金环游世界。

当然羊毛出在羊身上，英国工资的税较高，如果我再去打第二份工，那么需要交纳第二份工作40%的税。不过我还是挺感谢英国政府的，至少如果我这种情况，在国内孩子肯定要交给父母帮忙养，然后累死累活，生活质量基本上比不上在英国现在的生活。

浅谈 Naive Bayes 算法在石头剪刀布游戏的应用 Applying Naive Bayes Algorithm in Rock-Sissor-Paper Game


Image Credit: pixabay.com

The robot player just plays randomly without any intelligence at all and I am planing to add the basic intelligence to it by applying the Naive Bayes algorithm.

Known the probability of P(A), P(B), and the condition probability of P(A|B), then we can compute the P(B|A). How to apply this to the rock-sissors-papers game?

  1. Assume the game has been going on for a few turns and the robot has collected enough sampling data (data set)
  2. We need to compute the P(rock), P(paper), and P(sissors) for both player and the robot.
  3. Assume the player had X, then we need to compute winning probability of P(scissor | X), P(paper | X) and P(rock | X)
  4. Pick the one has the largest probability.

其中算法部分，我打算用 Naive Bayes 算法，这是一个很简单的概率型模型，简单来说就是:

已知 概率事件 P(A), P(B) 和条件概率 在B发生后A的概率 P(A|B)，那么我们能求出 在 A下 B的概率 P(B|A)

怎么应用到剪刀石头布上呢？很简单。

  1. 假设机器人已经收集到了足够的样本，也就是说玩家已经和机器人玩了 10局左右。
  2. 统计出 玩家和机器人 出 P(剪刀)、P(石头)和P(布) 分别的概率。
  3. 假设最后一局玩家出的是 X，那么需要算出 机器人 P(剪刀|X)、P(布|X)、P(石头|X)的胜率。
  4. 然后取最大的那个胜率出即可。

效果如何呢？让我们拭目以待吧。
