Too many versions bumping out in short time usually means that the QA process needs to be improved!

My backup node was replaying from 0.20.3 and guess what, there is a new version 0.20.5 today. Luckily, it doesn’t actually require a replay. Wow - First time jumping two versions! from 0.20.3 to 0.20.5!

  1. To fix these known issues for most accounts, witnesses should upgrade immediately without replaying.
  2. Witnesses should replay the blockchain to retroactively fix RC of all affected accounts.

So, I will do a full replay on my main server but at the meantime the backup server is online producing blocks using version 0.20.5!

Witness Page: https://helloacm.com/tools/steemit/witness-data/justyy

What have we learned?

Too many versions bumping out in short time usually means that the QA process needs to be improved! I seriously don’t want 0.20.6 tomorrow.

I cast a 100% vote and what happens?

HF20, what is new?

We had been expecting HF20 for a long time - but now the HF20 is here, we kinda are a little bit panic, so what is all about HF20? The most obvious change that HF20 brings is introducing mana and resources credit (RC). The previous voting power (VP) has been depreciated.

One example of 100% vote

I cast a 100% vote when voting mana is 60.66% and it reduces to 59.45% afterwards. Surprising, the RC isn’t used - but a slight increase due to automatic restore.

Max mana isn’t changed. Current mana is reduced from 64045630712730 to 62776518621168 and the difference is 1269112091562 - which is about the scale of approximate costs for a comment i.e.. 14200000000 - considering the voting mana is different.

We also notice that the cost for voting and transfer/powerups are similar - which are around 1/5 of commenting (posting).
