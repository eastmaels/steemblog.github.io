這件事可能比較敏感一點，所以我就不用英文了。

前一陣子，那真狼先生非常熱情地把Steem上許多App都整理出來，做了個美觀的網站，我還推薦過給大家。後來，也許是在他努力之下，有一些關於Dapp的網站 把咱們這些SteemApps都給列上去了，感覺是個很好的宣傳與曝光嘗試，大家都蠻開心的… 這也是因為我們確實有一些還不錯的Apps啊… 讓更多人知道是好事…

不過，其實我心中一直有個疑惑，其實，更早之前，約莫四個月前我也提出過，還給了一些想法：

從大屁屁到阿屁屁的

上面一篇文章我引述的Dapp定義，感覺太複雜了點。剛好最近我聽說了一個網站是專門列Dapp的，據說很出名，所以我就去看了一下：DappRadar.com，然後沒看見任何SteemApps（能被列成為主要平台的有三個，分別是Ethereum, EOS跟TRON），所以我就去找了他們關於Dapp的定義，指向這一篇文章：

WTF is an Ethereum Dapp ?

我覺得這一篇的簡單定義很乾淨清楚：

DApp can be a website, mobile app or something else. As long as the code responsible for main app logic is running on blockchain, it is a DApp.

嗯，我終於感覺比較能抓住這東西的重點了….

不過，這樣一來，其實Steem上面，應該是永遠也不能有Dapps了吧？至少現在以及甚至SMT想像中出來之後？

我去看了一遍，那真狼整理的目前列入Steem的Dapp網站，約有三四個…. 稍微看過之後，與上面那個網站比較，不得說，DappRadar.com感覺是最專業的。而他們並未認可SteemApps是Dapp（當然也有可能是還沒有機會…）。看過他們的定義之後，我覺得真的去提交也應該是沒有機會的。

其實，如果你夠了解區塊鏈世界，就會知道，DPOS系的東西，常常被批評不夠中心化。甚至有份報告（不過，這份報告的動機有被質疑就是了），說EOS並不是一個區塊鏈（為何針對EOS？因為人家市值夠大啊….）。不過，EOS至少有這領域喜歡的智能合約這東西。Steem其實，既有EOS的不夠去中心化批評（只是不夠大，人家不太注意），又沒有智能合約（描述事實，不是說我們應該要有）… 說真的，其實在區塊鏈界應該是一直比較處於邊緣的地位。這些話，在這裡不容易看到，但我心裡，其實一直是有些擔憂的….

常常在想，Steem的優勢，會不會只是先行者這一項而已？如果現在重新來過，要打造一個全新的Steem，應該用什麼架構？跟現在又會有多少不同呢？太難了…. 畢竟我也不是很懂區塊鏈的結構啊….

說遠了，這話題可以下次再聊。
