Steem Handbook Newsletter |《Steem 指南》快讯：校对组 + 融入社区组 + 催稿 + 上传状况查询 + 副主编 @maiyude 走马上任 |

This post is a contribution to the open source project Steem Handbook. Details could be found in the post Steem Handbook Calls for Support.

距离上回发布快讯有一个月了。币值下跌，《steem 指南》发起时的轰轰烈烈在慢慢冷却，每周进展不温不火。这才是真正考验的时候——很多项目都栽到这一步，烂尾了。《steem 指南》会不会烂尾？

将一个月的工作汇总起来，发现依然是可歌可泣的。除了新文稿的编写、发布、上传和编译外，我们看看还有哪些大事。

1. 校对组

在经过一番招兵买马后，校对组成功扩编，目前的成员为 @meixia @ivysrono @wilburm @matihe @intmilanmilano @bluestarpin。

2. 融入社区组

融入社区组对该章的任务进行了分配，并建立了内部讨论群，成员有 @wilhb81 @woolfe19861008 @wilburm @wickedgoose （全是 W），并且已经完成了部分章节的写作和上传。

3. 写作规范的完善

@wilburm 和 @dapeng 就前期的写作情况总结了经验和问题，对写作规范提出了看法并进行了完善，见这里和这里。

4. 催稿

很多作者在 steem 上发布稿件后，迟迟不向书稿的 github 项目提交。没提交就不入书，这等于没有参加本书的写作。

编辑部讨论了一下可能的原因：

  • github 的使用有一定门槛，不容易学；
  • 文稿没能按书稿要求完成修订；
  • 春节期间忙着度假，把交稿给忘了；
  • 提交这件事本身是没有收益的。

为此，卖鱼的奶爸 @maiyude 将书稿通读了一遍，在已提交和未提交的发帖后面分别留言，催促大家尽快交稿。

作者们如存在技术困难，例如 github 的使用等，请向编辑部反馈，一起想办法解决。

5. 提交状况查询

为了方便查看哪篇帖子以提交， @dapeng 对书稿查询工具 steemr.org 进行了升级，帖子列表里新增 “github”一列，凡是画 x 的，都是已提交的。

如何判断已提交？凡是 @maiyude 留言“本文已收进steemh.org”的，就是已提交的。

以后，请作者们提交到 github 项目后告知 @maiyude，由他审核后留言，steemr 会在他留言后自动更新提交状态。

6. 邀请 @maiyude 担任副主编

卖鱼的奶爸 @maiyude 对 《steem 指南》的贡献有目共睹，除了上面的催稿外，还有：设计封面，logo，各种周边产品；编写了贯穿全书的小故事；参与了几个重要章节的编写工作；帮助一些作者上传文稿，在编辑部的讨论里提出了很多有益的建议……

我们邀请 @maiyude 担任本书的副主编。撒花，鸣炮，奏乐。
